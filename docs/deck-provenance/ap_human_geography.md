## Pool Design Rationale

### Architecture evolution
- 19 initial pools → 36 final semantic pools after redesign
- Key principle: every pool must pass the "plausible distractor test" — can every member serve as a plausible distractor for every other member's question?
- Junk-drawer splitting pattern documented here as a reusable template (see "Reusable Patterns" below)

### Final Pool Groups

**Unit-Specific Concept Sub-Pools (replacing junk drawer)**
- `unit1_concept_definitions` — map types, scale, spatial concepts
- `unit2_population_terms` — population metrics, demographic concepts
- `unit2_migration_terms` — migration types, push/pull factors
- `unit3_culture_terms` — cultural diffusion, cultural landscape
- `unit3_religion_language_terms` — religious traditions, language families (split from `unit3_culture_terms` after two-right-answer bug)
- `unit4_political_terms` — sovereignty, boundary classification
- `unit5_agriculture_terms` — farming systems, agricultural models
- `unit6_urban_terms` — urban models, land use zones
- `unit7_development_terms` — economic development vocabulary
- ... (additional unit-specific pools)

**Cross-Unit Named-Entity Pools**
- `model_and_theory_names` — all named models (DTM, Von Thunen, Burgess, Hoyt, etc.)
- `urban_model_names` — urban structure models only (subset enabling targeted questions)
- `language_family_names` — explicitly separated to prevent answer ambiguity
- `economic_indicator_names` — GDP, GNI, HDI, GII, etc.
- `development_theory_names` — Rostow, Wallerstein, dependency theory
- `migration_type_names` — migration categories and legal statuses
- `boundary_type_names` — political boundary classification
- `diffusion_type_names` — cultural diffusion taxonomy
- `agriculture_practice_names` — farming systems only
- `economic_sector_names` — primary through quinary (5 facts, `homogeneityExempt: true`)
- `state_shape_names` — exactly 5 by definition (`homogeneityExempt: true`)

**Numeric Pool**
- `bracket_numbers` — DTM stages, Rostow stages, key dates, unit counts, exam percentages

## Distractor Strategy

- **Per-fact distractors**: LLM-generated at creation time, reading the specific question
- **Runtime pool distractors**: Pool-based selection from semantically-coherent unit sub-pools
- **Confusion matrix**: Weights toward previously confused answers after player makes errors
- Pool homogeneity enforced so runtime distractors are always semantically plausible

## Reusable Patterns

This deck established several production patterns now recommended for all AP-exam decks:

1. **Pool-first architecture**: Define all answer type pools before generating a single fact. Prevents the "junk drawer" problem where a catch-all definition pool produces eliminatable distractors.

2. **Unit sub-pool splitting**: Never share a `concept_definitions` pool across multiple AP units. Unit 1 and Unit 6 vocabulary will inevitably cross-contaminate. Name pools `unit{N}_{type}` from the start.

3. **Language/language-family separation**: `language_family_names` (Indo-European, Sino-Tibetan) and `language_type_terms` (creole, pidgin, lingua franca) must be in separate pools — both can answer "type of language" questions, causing two-right-answer failures.

4. **Fill-blank rewrite**: Knowledge decks must not use `{___}` fill-in-blank question format. Convert all `{___}` questions to direct format ("What is the term for...?" style) before QA.

5. **Curriculum-sourced scope**: Every AP fact must trace to a specific EK code. Workers without EK citations invented incorrect details (e.g., wrong stage counts, fabricated model names). Always provide the EK codes in the generation prompt.
