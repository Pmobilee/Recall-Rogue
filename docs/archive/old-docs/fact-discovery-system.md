# Fact Discovery System — Design Spec

## v2 Revision — "Free First Charge" (2026-03-17)

> **STATUS: The design below (overlay + cardHint) has been SUPERSEDED. The new implementation is "Free First Charge." See AR-59.23 phase doc for the current spec.**

The original design described here — discovery overlay on first Quick Play, `cardHint` hint text on cards, amber/golden glows — was scrapped before implementation. The core problem: **card avoidance behavior.** Showing visible hint text on some cards and not others creates a rational signal for players to avoid Charging unknown cards ("no hint = risky"). This is the opposite of the educational goal.

**Replacement design in brief:**
- First Charge of any fact in a run = FREE (0 AP surcharge, 1.0× on wrong = same as Quick Play, full multiplier on correct)
- The CHARGE button shows "FREE" instead of "+1 AP" — the only per-card indicator
- No cardHint field, no discovery overlay, no card-level visual states
- Tracked via `firstChargeFreeFactIds: Set<string>` in RunState
- After the free Charge is used, fact behaves normally (+1 AP, 0.7× on wrong)

The run arc (exploration → mastery) is preserved: Act 1 most buttons say "FREE", Act 3 most say "+1 AP". But without per-card visual stigma that causes avoidance.

The original design spec is preserved below for historical reference only.

---

## Overview

The Fact Discovery System solves the cold-start problem for the Charge mechanic: paying +1 AP to Charge a card you've never seen before is a blind guess that will almost always fail. Instead of punishing new players with wasted AP, the system introduces a **preview-before-test** flow that rewards natural play progression.

**Core rule: The QUESTION is visible. The ANSWER is never visible.** Players know what they'll be asked, but must still retrieve the answer from memory. This is priming, not cheating.

## The cardHint Field

Every fact needs a short topic prompt that fits on a card's parchment area below the mechanic text. Full quiz questions are too long (8-15 words). The `cardHint` field is a 2-5 word condensed prompt.

### Knowledge Facts

Generated by Haiku during ingestion. One extra field per fact.

| Full Quiz Question | cardHint |
|---|---|
| How much blood can a leech consume relative to its body weight? | Leech blood intake? |
| How many eyelids does a camel have? | Camel eyelids? |
| How often do Canada lynx and snowshoe hare populations peak? | Lynx-hare cycle length? |
| Who published the Surrealist Manifesto in 1924? | Surrealist Manifesto author? |
| What makes Venus's rotation unusual compared to most planets? | Venus rotation? |
| What is most sushi restaurant 'wasabi' actually made from? | Fake wasabi ingredient? |
| Which emperor ordered the initial construction of the Great Wall? | Great Wall emperor? |
| What percentage of rice is milled away for daiginjo sake? | Daiginjo rice milling? |
| What was the first device that could record and play back sound? | First sound recorder? |
| In what year did Newton publish the Principia Mathematica? | Principia year? |
| Which DNA base always pairs with adenine (A)? | Adenine's base pair? |
| What did Hubble prove in 1923 by observing Cepheid stars? | Hubble's Andromeda proof? |
| Which deity is Adi Shankara traditionally considered an avatar of? | Shankara's deity? |
| What unusual process causes peanut pods to develop underground? | Peanut underground process? |

**Rules for cardHint generation:**
- 2-5 words maximum
- Always ends with `?`
- Names the core subject + what's being asked
- Never contains the answer
- Never ambiguous — player should recognize "oh, THAT question" after seeing the full fact once

### Vocabulary Facts

No generation needed. The cardHint IS the target word, already in the data.

| Language | cardHint | Source Field |
|---|---|---|
| Japanese vocab | 食べる → ? | `targetWord` |
| Japanese kanji | 影 → ? | kanji character from question |
| Japanese grammar | について → ? | grammar pattern from question |
| Korean | 상주 → ? | `targetWord` |
| Chinese | 黑心 → ? | `targetWord` |
| Spanish | encima → ? | `targetWord` |
| French | éthique → ? | `targetWord` |
| German | privat → ? | `targetWord` |
| Dutch | aanwezigheid → ? | `targetWord` |
| Czech | napsat → ? | `targetWord` |

For vocabulary, the arrow notation (`→ ?`) signals "what does this mean?" without needing to write the full question.

### Data Model Addition

```typescript
interface Fact {
  // ... existing fields ...
  cardHint: string;  // 2-5 word topic prompt, never contains answer
}

interface VocabularyFact extends Fact {
  // cardHint auto-derived: `${targetWord} → ?`
}
```

### Pipeline Generation

For knowledge facts, add to the Haiku processing prompt:

```
Generate a cardHint for this fact: a 2-5 word topic prompt ending 
with "?" that names the subject and what's being asked. Must NOT 
contain the answer. Must be recognizable after seeing the full 
question once.

Question: "How many eyelids does a camel have?"
Answer: "Three"
cardHint: "Camel eyelids?"
```

Batch cost estimate: ~$2 for 100K facts at Haiku rates. Single pipeline run.

For vocabulary facts, cardHint is computed at build time:

```typescript
function deriveVocabCardHint(fact: VocabularyFact): string {
  return `${fact.targetWord} → ?`;
}
```

## Discovery Flow

### First Appearance in a Run

When a fact appears for the **first time in the entire run** (not per-encounter — per-RUN), and the player Quick Plays it:

1. Card plays normally at 1.0x (Quick Play speed, 200ms)
2. During the effect animation, a semi-transparent overlay briefly shows the **full fact statement**:
   > *"A camel has three eyelids — the third is a translucent membrane for clearing dust"*
3. Overlay appears for 1.0-1.5 seconds, fades automatically
4. The factId is added to `runDiscoveryState.discoveredFactIds`
5. All future appearances of this fact during the run show the `cardHint` on the card

**The overlay shows the full fact (question + answer + brief context) — this is the ONE time the answer is visible.** This is the learning moment. After this, only the question hint is shown, and the player must recall the answer from memory.

### Subsequent Appearances in the Run

After discovery, every card that draws this fact shows the `cardHint` in the parchment area:

```
┌─────────────────────┐
│     Strike          │  ← Mechanic name
│                     │
│   [Card Art]        │
│                     │
│  Deal 8 damage      │  ← Mechanic effect
│  ─────────────      │
│  Camel eyelids?     │  ← cardHint (discovered)
└─────────────────────┘
```

Undiscovered cards show only the mechanic text:

```
┌─────────────────────┐
│     Strike          │
│                     │
│   [Card Art]        │
│                     │
│  Deal 8 damage      │
│                     │  ← No hint (unknown fact)
│                     │
└─────────────────────┘
```

The visual difference is immediately obvious at a glance: cards with text below the line = discovered (safe to Charge). Cards without = unknown (risky to Charge).

### Discovery Overlay UX

| Phase | Duration | What Happens |
|---|---|---|
| Quick Play tap | 0ms | Card effect triggers instantly |
| Effect animation | 0-200ms | Damage numbers fly, enemy reacts |
| Overlay fade-in | 200ms | Semi-transparent dark panel appears at bottom of combat area |
| Fact visible | 1000-1200ms | Full fact statement displayed (question + answer + brief context) |
| Overlay fade-out | 300ms | Panel fades. Card is now marked as discovered. |
| Total added time | ~0ms | Overlay is concurrent with existing animations, not sequential |

**Visual design of overlay:**
- Semi-transparent dark panel (rgba black, ~70% opacity)
- Fact text in white, 14-16px, centered
- Brief — covers the essential fact, not the full explanation
- Positioned at bottom of combat area, above the card hand
- Subtle "DISCOVERED" or lightbulb icon in corner
- Does NOT block enemy sprite or card interactions

## Visual States

### Card Bottom Edge Glow

| Run State | Parchment Shows | Bottom Glow | Charge Risk |
|---|---|---|---|
| **Unknown** (first time in run, not yet Quick Played) | Mechanic text only | None | Blind — don't know what you'll be asked |
| **Discovered** (Quick Played once this run) | Mechanic text + cardHint | Warm amber | Informed — you know the question, must recall answer |
| **Tier 2+ Proven** (answered correctly in past runs) | Mechanic text + cardHint (auto-discovered) | Warm amber from first appearance | Low — you've answered this before |
| **Tier 3 Mastered** | Mechanic text + "MASTERED" | Golden | Zero — auto-Charges at 1.2x, no quiz |

### Card Visual Layer Stack (Bottom to Top)

1. **Card frame PNG** — per-mechanic, existing asset
2. **Chain color tint** — left edge, 2-3px colored line (categoryL2 indicator)
3. **Discovery glow** — bottom edge, warm amber (fact knowledge state)
4. **AP gemstone badge** — top-left corner (energy cost, colored by AP: green/blue/orange/red)
5. **Surge glow** — golden shimmer overlay (Knowledge Surge turns only, full card)
6. **Card content** — mechanic name (top), card art (middle), parchment area (bottom: effect text + cardHint)

### Hand Readability at Fan Overlap

In the card fan, only the top-left and bottom portions of each card are reliably visible. The system is designed so that:
- **Chain tint** (left edge) is visible even on heavily overlapped cards
- **Discovery glow** (bottom edge) is visible on the bottom card area
- **cardHint text** is in the parchment area which is the LEAST overlapped part of the fan (bottom cards push out below the overlap)
- **AP badge** (top-left) is always visible

Players can tap any card to pop it up for full inspection, where all elements are clearly visible.

## Per-Run Tracking (NOT Per-Encounter)

### Why Per-Run, Not Per-Encounter

**Per-encounter (rejected):** Facts reset to Unknown every fight. Player sees same facts as Unknown repeatedly. No investment. Player thinks "I'll just get different facts next encounter" and ignores hard ones.

**Per-run (chosen):** Once discovered, a fact stays discovered for the entire run. The same hard question keeps appearing on different cards across encounters. After seeing "Surrealist Manifesto author?" on 4-5 different cards, the player has SEEN the question repeatedly, building familiarity. When they finally decide to Charge it and recall "André Breton," that's a genuine learning moment earned through repeated exposure.

Per-run tracking also means:
- A 25-minute run with 120 facts in the pool and ~60 drawn across all encounters will discover ~30-40 facts through natural Quick Play
- By Act 3, most of the player's hand shows cardHints — making strategic Charging increasingly viable as the run progresses
- The natural arc: Act 1 = exploratory (lots of unknowns), Act 2 = transitional (mix of known and unknown), Act 3 = powerful (mostly known, Charge confidently)

### Data Model

```typescript
interface RunDiscoveryState {
  discoveredFactIds: Set<string>;  // Facts Quick Played at least once this run
}

// Check if a fact should show cardHint and warm glow
function isDiscovered(
  factId: string, 
  fsrsTier: number, 
  runState: RunDiscoveryState
): boolean {
  if (fsrsTier >= 2) return true;           // Tier 2+ always discovered
  return runState.discoveredFactIds.has(factId);  // Tier 1: needs Quick Play
}

// On Quick Play of a Tier 1 fact, mark as discovered
function onQuickPlay(
  factId: string, 
  fsrsTier: number, 
  runState: RunDiscoveryState
): boolean {
  if (fsrsTier >= 2) return false;  // Already discovered, no overlay needed
  if (runState.discoveredFactIds.has(factId)) return false;  // Already discovered this run
  
  runState.discoveredFactIds.add(factId);
  return true;  // Return true = show discovery overlay
}
```

### Save/Resume Integration

`discoveredFactIds` is included in the run save state (it's just a Set of strings, negligible size). If a player saves mid-run and resumes, their discovery progress is preserved.

### Reset

The discovery set is cleared when:
- Run ends (victory, defeat, abandon)
- New run starts

It is NOT cleared between encounters, between acts, or on rest site visits.

## Interaction with Other Systems

### Knowledge Chains

Chain indicators (left-edge color tint) are **always visible regardless of discovery state.** A player can see that two cards chain together (matching colors) even if both are Unknown. This creates a decision: "These two chain, but I haven't discovered either. Do I Quick Play both to discover their questions (losing chain bonus this turn) or Charge blind for the chain multiplier?"

### Knowledge Surge

During Surge turns (Charge costs +0 AP), Unknown cards are still risky to Charge because the player doesn't know the question. Surge makes Charging **free** but not **safe**. Discovery still matters on Surge turns — the strategic play is to Charge discovered cards during Surge and Quick Play unknowns.

### Boss Quiz Phases

Boss quiz phases present questions directly — no cards involved, no discovery system. The questions draw from the run's fact pool. However, facts that were discovered during regular combat may appear in boss quiz phases, and the player has the advantage of having seen them before. This creates a subtle reward for thorough discovery: "I Quick Played a lot of diverse facts in Acts 1-2, so I recognize more questions in the Act 3 boss's rapid-fire phase."

### Rest Site Study

Study questions at rest sites draw from the run's fact pool. If a fact was already discovered during combat, the player has an advantage in Study. Facts answered correctly during Study do NOT count as "discovered" for the card display (Study doesn't Quick Play a card), but they DO update FSRS state — so a correct Study answer on a Tier 1 fact may promote it to Tier 2a, at which point it auto-discovers in future runs.

### Echo Cards

Echo cards (wrong-answer ghosts that reappear later) **always show as Discovered** with the cardHint visible. The player already saw the question when they failed the original Charge — that's the educational purpose of echoes. They know what they got wrong. The hint on the echo card says "try again."

### Shop Haggling

Shop haggle questions draw from the run's fact pool. Discovery state doesn't apply to shop quizzes (no card involved), but the same familiarity advantage applies — discovered facts are more likely to be correctly answered.

### Onboarding (Run 1)

During Run 1 tutorial:
1. First Quick Play: "You discovered a new fact! Facts you've seen will appear as hints on your cards."
2. Next hand with a discovered card: "See the hint below the card effect? You've seen this fact before. Drag up to Charge it — answer correctly for 3× power!"
3. This naturally teaches the discovery → Charge flow without explicit rules explanation.

## Edge Cases

| Scenario | Behavior |
|---|---|
| Same fact appears in multiple hands within one encounter | Discovered after first Quick Play. All subsequent appearances in the run show cardHint. |
| Player Charges an Unknown card (blind guess) | Allowed. Quiz appears normally. If correct: FSRS records success AND fact is marked discovered. If wrong: 0.7x effect, AP wasted. Natural punishment. |
| Player Charges an Unknown card and gets it RIGHT | Fact is now discovered (shows cardHint on future appearances). Player both learned the fact AND got the power bonus. High-risk, high-reward. |
| Fact is Tier 1 but was discovered in a previous encounter this run | Still Discovered (per-run tracking). cardHint still visible. |
| Fact is Tier 2+ but player hasn't played it yet this run | Starts as Discovered (cardHint visible) from first appearance. FSRS confirms prior knowledge. |
| Fact is Tier 3 (Mastered) | Shows "MASTERED" instead of cardHint. Golden glow. Auto-Charges at 1.2x. No quiz. |
| Two cards have the same fact (possible if draw pile reshuffles) | Both show the same cardHint after discovery. Both are equally safe to Charge. |
| Discovery overlay fires during a chain | Overlay appears as normal. Chain multiplier still applies to the Quick Play damage. Discovery is visual feedback, not a gameplay interrupt. |
| 120-fact pool, all Tier 1 (new player, new domain) | First run in a new domain = lots of unknowns. By end of Act 2, ~25-35 facts discovered through natural Quick Play. Act 3 hand is mostly discovered. This IS the intended pacing. |
| 120-fact pool, all Tier 2+ (veteran player, familiar domain) | Every card shows cardHint from the start. Player can Charge confidently from turn 1. This IS the reward for learning. |

## Strategic Implications

### The Discovery Arc Within a Run

**Act 1 (The Shallows):** Most cards are Unknown for new domains. Player Quick Plays frequently, discovering facts. Charging is occasional and risky. Combat is manageable with Quick Play alone.

**Act 2 (The Depths):** ~50-60% of encountered facts are now Discovered. Player sees familiar cardHints and starts Charging strategically. The Examiner elite forces at least some Charging — discovered cards make this survivable.

**Act 3 (The Archive):** ~70-80% of encountered facts are Discovered. Hand is mostly hints and glows. Player Charges confidently and frequently. The Librarian elite (immune to Quick Play) is beatable because the player has a run's worth of discovery.

### Knowledge Confidence as a Resource

Discovery creates a new invisible resource: **knowledge confidence.** A hand with 4 discovered cards and 1 unknown is a strong hand — you can Charge 1-2 of the known ones safely. A hand with 1 discovered and 4 unknown is a weak hand — you're mostly Quick Playing.

This means **deck composition affects knowledge confidence.** A player studying Japanese N5 (800 facts, many repeated) will see more repeats per run and discover facts faster. A player doing "All Topics" across 10 domains (10K+ facts) will see more unknowns per run. This naturally makes focused study decks feel more powerful within a run (faster discovery, more Charging) while broad decks offer more variety but slower discovery.

This is actually the correct educational tradeoff: focused study = faster mastery = more power. Broad study = slower mastery = more learning breadth. Both are valid, and the game rewards both differently.

### Metacognitive Skill Development

The cardHint system explicitly trains metacognition — the ability to assess your own knowledge. Every time a player sees "Camel eyelids?" and thinks "do I know this?", they're practicing self-assessment. Players who accurately predict their own knowledge (Charge facts they actually know, Quick Play facts they don't) perform better. Overconfident players waste AP on wrong Charges. Underconfident players leave power on the table.

This is a real educational skill. Dunning-Kruger research shows that poor performers systematically overestimate their knowledge. The Charge system provides immediate, tangible feedback on metacognitive accuracy — wrong Charges HURT, right Charges REWARD. Over time, players calibrate their self-assessment, which transfers to real-world learning.

## Summary

The Fact Discovery System transforms the Charge mechanic from a blind gamble into a strategic knowledge-assessment game:

1. **First time a fact appears in a run:** Quick Play shows full fact as discovery overlay (1.5s). Player learns the fact exists.
2. **All subsequent appearances:** Card shows `cardHint` in parchment area (2-5 word question prompt). Player knows what they'll be asked but must recall the answer.
3. **Charging a discovered card:** Player assesses "do I know this?" and decides whether the +1 AP risk is worth 3.0x power.
4. **Tier 2+ facts:** Auto-discovered from first appearance (FSRS confirms prior knowledge).
5. **Tier 3 facts:** Auto-Charged, no quiz, permanent 1.2x power. The endgame reward.

The system creates a natural run arc from exploration (Quick Play unknowns) to power (Charge discovered facts), trains metacognitive self-assessment, and makes knowledge mastery visually tangible in the glow pattern of every hand.
