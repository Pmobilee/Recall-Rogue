# AR-59.10 — Five-Slot Relic System

**Status**: Complete
**Complexity**: Medium
**Dependencies**: None at code level. AR-59.11 (Relic Catalogue) defines which relics exist; AR-59.12 (Acquisition Flow) defines when they are offered.

---

## Overview

Change the relic system from unlimited collection (`runRelics: Array<...>` with no cap) to a hard **5-slot limit**. When a player reaches 5/5 relics and a new one is offered, they must either sell an equipped relic to make room or pass on the new one entirely. A single cursed rare relic — Scholar's Gambit — expands the cap to 6.

Additional D-series decisions woven into this AR:

- **D14** — Mastery Coins persisted. **25 relics unlock at run start** (all Commons + some Uncommons). Remaining **~17** (Rares + Legendaries) require Mastery Coin purchases from the Hub Relic Archive. `isStarter` field on `RelicDefinition` already exists — a new `startsUnlocked: boolean` flag supplements it to express the "25 free" pool vs "coin-required" pool distinction.
- **D15** — Relic reroll added at boss/mini-boss relic selection: spend **50g** to reroll all 3 options once per selection. Merchant's Favor relic gives 4 choices instead of 3 (separate concern, noted here for context).
- **D16** — Relics that reference unbuilt mechanics are **excluded from Phase 1 run pool** via a new `excludeFromPool: boolean` flag on `RelicDefinition`. Specifically: Toxic Bloom (requires multi-enemy mechanic) and any others identified.

The **starter relic selection screen** is **removed**. Relics are acquired exclusively through encounters (boss rewards, mini-boss rewards, merchant purchases, shrine drops). Runs begin with 0 relics in slots.

## Design Decisions Summary

| Item | Current | Target |
|------|---------|--------|
| Relic limit | Unlimited | 5 slots (6 with Scholar's Gambit) |
| Acquisition | Unlimited collection | Choose carefully; sell to make room when full |
| Mastery Coins | Buy any relic from Hub | 25 free at run start; ~17 require coins |
| Relic reroll | N/A | 50g to reroll choices once per selection event |
| Starter relic | Choose 1 of 3 fixed starters | **REMOVED** — relics from encounters only |
| Sell refund | N/A | 40% of relic's rarity-based shop price (rounded down) |

### Step 7 — Add Sell Helper to `gameFlowController.ts`

**File**: `src/services/gameFlowController.ts`

Add:

```typescript
import { RELIC_SELL_REFUND_PCT, SHOP_RELIC_PRICE } from '../data/balance';
import { RELIC_BY_ID } from '../data/relics';

/**
 * Sells an equipped relic mid-run, refunding 40% of its rarity-based shop price.
 * Removes the relic from runRelics and adds gold to the run.
 * Returns the gold refunded, or 0 if the relic was not found.
 */
export function sellEquippedRelic(definitionId: string): number {
  const run = get(activeRunState);
  if (!run) return 0;

  const idx = run.runRelics.findIndex(r => r.definitionId === definitionId);
  if (idx === -1) return 0;

  const def = RELIC_BY_ID[definitionId];
  const basePrice = def ? (SHOP_RELIC_PRICE[def.rarity] ?? 60) : 60;
  const refund = Math.floor(basePrice * RELIC_SELL_REFUND_PCT);

  run.runRelics.splice(idx, 1);
  run.gold += refund;
  activeRunState.set(run);
  return refund;
}
```

Sell refund amounts:
- Common (60g × 0.40) = **24g**
- Uncommon (100g × 0.40) = **40g**
- Rare (160g × 0.40) = **64g**
- Legendary (250g × 0.40) = **100g**

**Acceptance criteria**: `sellEquippedRelic` exported. Calling it removes the relic and adds the correct gold. `npm run typecheck` passes.

### Step 9 — Relic Reroll State

**File**: `src/services/gameFlowController.ts`

Add module-level reroll tracking:

```typescript
/** Rerolls used in the current relic selection event. Reset each time a new selection opens. */
let currentSelectionRerollsUsed = 0;

/** Reset reroll counter — call when opening a new boss/mini-boss relic selection screen. */
export function resetRelicSelectionRerolls(): void {
  currentSelectionRerollsUsed = 0;
}

/**
 * Returns true if the player can reroll the current relic selection.
 * Requires: not yet rerolled this selection AND gold >= RELIC_REROLL_COST.
 */
export function canRerollRelicSelection(): boolean {
  const run = get(activeRunState);
  if (!run) return false;
  return currentSelectionRerollsUsed < RELIC_REROLL_MAX && run.gold >= RELIC_REROLL_COST;
}

/**
 * Rerolls the current relic selection. Deducts 50g, marks reroll used,
 * marks current offerings as seen, draws 3 fresh options.
 * Returns new relic IDs, or null if not allowed.
 */
export function rerollRelicSelection(currentOfferedIds: string[]): string[] | null {
  const run = get(activeRunState);
  if (!run || !canRerollRelicSelection()) return null;

  run.gold -= RELIC_REROLL_COST;
  currentSelectionRerollsUsed++;

  // Mark the rerolled options as seen so they can't re-appear
  currentOfferedIds.forEach(id => run.offeredRelicIds.add(id));

  // Draw fresh options from eligible pool (excludes equipped + previously offered)
  const newOptions = drawRelicOptions(run, RELIC_BOSS_CHOICES);
  activeRunState.set(run);
  return newOptions.map(r => r.id);
}
```

Verify that `drawRelicOptions` (or equivalent pool-draw function) already filters by `offeredRelicIds` and respects `excludeFromPool`. If it does not, add those filters here.

**Acceptance criteria**: Reroll deducts 50g, increments counter, returns fresh options excluding seen relics. Second reroll attempt returns null. `npm run typecheck` passes.
