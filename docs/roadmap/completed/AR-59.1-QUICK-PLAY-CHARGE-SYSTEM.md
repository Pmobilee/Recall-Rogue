# AR-59.1 — Quick Play / Charge System

**Status:** Pending
**Depends on:** Nothing — this is the root dependency for AR-59.x
**Blocks:** AR-59.2 (Fling Gesture), AR-59.3 (Chain Reward), AR-59.4 (Surge Turns)

---

## Overview

Replace the current quiz-on-every-card-play model with a two-mode system:

- **Quick Play:** Tap popped card a second time → instant play at 1.0× base power, no quiz, no AP surcharge
- **Charge Play:** Tap the floating **CHARGE!** button (or fling up, AR-59.2) → quiz triggers, +1 AP surcharge, multiplier depends on quiz result and card tier

This is the single most important change in the v2 overhaul. Every other sub-AR (chains, surge, gesture) depends on the card play event including `playMode` metadata. Do not implement those sub-ARs before this one.

### Design Decisions (locked — do not re-litigate)

| # | Decision |
|---|----------|
| D1 | Keep two-tap flow: tap to inspect → tap CHARGE or tap card to Quick Play |
| D2 | Tier 3 cards auto-Charge at 1.2× with no quiz and no +1 AP surcharge (strictly better, by design) |
| D4 | Wrong Charge = 0.6×–0.7× depending on tier (generous — player already paid +1 AP) |
| D5 | Quick-Play-only is viable in Act 1, tight in Act 2, near-impossible in Act 3 (enforced mechanically + tuning) |

---

## Tier-Based Charge Multiplier Table

| Tier | SM-2 Display | Quick Play | Charge Correct | Charge Wrong | Charge AP Cost |
|------|-------------|------------|----------------|--------------|----------------|
| `'1'` | Learning | 1.0× | 2.5× | 0.6× | base + 1 AP |
| `'2a'` | Proven | 1.0× | 3.0× | 0.7× | base + 1 AP |
| `'2b'` | Proven+ | 1.0× | 3.5× | 0.7× | base + 1 AP |
| `'3'` | Mastered | 1.2× (auto) | Auto-Charged (no quiz) | N/A | base AP only (free) |

Tier `'3'` cards are resolved as Charge-correct at 1.2× automatically. The CHARGE button is hidden for Tier 3 cards. No quiz fires.

## Current Implementation (as of AR-59.1 start)

Understanding the current code is essential. Read this section carefully.

### Current Card Play Flow

1. **`hand` stage** — cards displayed in hand row
2. **`selected` stage** — player taps a card → `selectedIndex` is set, card pops up
3. **`committed` stage** — player taps the popped card again (or taps "Cast" button) → quiz appears immediately via `handleCast()` in `CardCombatOverlay.svelte`
4. **Quiz answered** → `handleAnswer(answerIndex, isCorrect, speedBonus)` calls `onplaycard(cardId, correct, speedBonus, responseTimeMs, variantIndex)`
5. **`onplaycard` prop** (set in `CardApp.svelte`) calls `playCardAction(turnState, cardId, answeredCorrectly, speedBonusEarned)` in `turnManager.ts`

### Current CardPlayStage Type (in `CardCombatOverlay.svelte`, line 58)

```ts
type CardPlayStage = 'hand' | 'selected' | 'committed'
```

The `committed` stage currently means "quiz is showing." After this AR, it will mean "quiz is showing for a Charge play." Quick Play bypasses the committed stage entirely.

### Current `playCardAction` Signature (`turnManager.ts`, line 304)

```ts
export function playCardAction(
  turnState: TurnState,
  cardId: string,
  answeredCorrectly: boolean,
  speedBonusEarned: boolean,     // ← REMOVE THIS PARAMETER
): PlayCardResult
```

The `speedBonusEarned` parameter is currently used to apply a 1.5× damage multiplier for fast answers (`SPEED_BONUS_MULTIPLIER = 1.5`). This entire system is removed in this AR.

### Current Wrong Answer Behavior (`turnManager.ts`, lines 396–467)

Currently, wrong answers apply a partial fizzle: `Math.round(card.baseEffectValue * FIZZLE_EFFECT_RATIO)` where `FIZZLE_EFFECT_RATIO = 0.25`. The card still "fires" at 25% power. The `fizzled: true` flag is set in `PlayCardResult`.

After this AR: wrong Charge applies the tier-based wrong multiplier (0.6×–0.7×) to the full card effect pipeline (same as correct, but lower multiplier). The `fizzled` flag is repurposed to mean "played wrong" and still fires the effect. Quick Play never fizzles.
