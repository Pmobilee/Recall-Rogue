# AR-59.16 — Charge Play & Quick Play Animation Polish

**Status:** Pending
**Depends on:** AR-59.1 (playMode field on card play events), AR-59.2 (fling gesture + drag infrastructure), AR-59.4 (Surge turn state)
**Estimated complexity:** Medium-High (CSS-heavy, animation orchestration branching, Phaser integration)

---

## Overview

Polish the Charge Play and Quick Play animations to make Charging feel powerful and deliberate while Quick Play feels lightning-fast. This is the "game feel" pass for the core combat interaction introduced in AR-59.1.

**Goal:** Differentiate the two play modes visually and haptically so that the choice between Quick Play and Charge Play carries felt weight. A Charged correct answer should feel like landing a critical hit. A Quick Play should feel instant and frictionless. A wrong Charge should feel disappointing but not punishing.

**What this is NOT:**
- This doc does not add new mechanics — it animates existing ones.
- It does not change damage values or multipliers (those live in AR-59.1/AR-59.8).
- It does not add new audio assets — it uses existing audio cues or no-ops where new sounds would go.

## Animation Specifications

### Quick Play Animation — 200ms total (FAST)

Quick Play must feel instant. No dramatic buildup. Snap → effect → done. The player chose speed; reward it with immediacy.

| Phase | Duration | Description |
|-------|----------|-------------|
| Snap | 50ms | Card scales to 1.05× briefly (CSS `scale`) |
| Effect | 100ms | Abbreviated mechanic-specific micro-animation (no flip, no swoosh) |
| Resolve | 50ms | Card slides to discard position and fades (opacity 0) |

**Rules:**
- No card flip (no `reveal` phase, no cardback shown)
- No `swoosh` or `impact` phases — replaced by inline CSS micro-animation
- Damage numbers appear at 1.0× size, standard white color
- Haptic: single light tap (`tapLight()` — add to haptic service if not present, else use `tapMedium()`)
- Sound: existing `card-discard` cue (short crisp)
- `juiceManager.fire()` still fires with `type: 'correct'` — damage number and enemy hit still trigger

**New `CardAnimPhase` value:** `'quick-play'`
**Duration constant:** `QUICK_PLAY_DURATION = 200`

### Charged Play — Correct Answer — 500ms total (POWERFUL)

Charging correctly should feel like landing a critical hit — more powerful than any existing animation.

| Phase | Duration | Description |
|-------|----------|-------------|
| Flash | 100ms | GREEN screen flash (use `pulseFlash(0x44FF44, 0.2, 100)`) + golden particle burst from card position |
| Power-up | 150ms | Card scales to 1.3× with golden `box-shadow` glow intensifying (CSS keyframe) |
| Impact | 150ms | Card launches toward enemy (existing `impact` phase behavior). Screen shake via `cameras.main.shake(100, 0.004)`. Full mechanic animation runs |
| Resolve | 100ms | `discard` phase. Damage numbers at 1.5× font size, golden color (`isCritical: true` flag handles this via existing `DamageNumber.svelte` styling, OR add new `isCharged: boolean` prop) |

**Rules:**
- Card flip to cardback IS used (keep `reveal` phase — charged plays get the full reveal)
- Full mechanic animation plays at existing intensity
- Haptic: `tapHeavy()` + delayed second `tapHeavy()` at +80ms (existing critical pattern)
- Sound: existing `correct-impact` or `correct-critical` cue
- GREEN flash (not white) — distinguish from speed-bonus blue-white flash
- Screen shake: routed through `juiceManager` → `onScreenFlash` callback → new `onScreenShake(durationMs, intensity)` callback (add to `JuiceCallbacks` interface)
- Particle burst: golden (`0xFFD700`), count 35 (slightly more than current 30)

**New `CardAnimPhase` value:** Not needed — Charge Correct reuses the existing `reveal → swoosh → impact → discard` sequence with enhanced `juiceManager` events. The `playMode: 'charge-correct'` context passed to `juiceManager.fire()` triggers enhanced feedback.

**Add to `JuiceEvent`:** `playMode?: 'quick' | 'charge-correct' | 'charge-wrong' | 'auto-charge'`

### Charged Play — Wrong Answer — 300ms total (MUTED)

Wrong Charge should feel disappointing but not punishing. Brief red tint, muted resolution.

| Phase | Duration | Description |
|-------|----------|-------------|
| Dim | 100ms | Subtle desaturation overlay (red tint, CSS `filter: saturate(0.6)` on `.card-hand-container`, not a harsh flash) |
| Reveal | 100ms | Correct answer shown in blue highlight (existing behavior — keep as-is) |
| Resolve | 100ms | Card plays at 0.7× opacity during slide-to-discard. Damage numbers at 0.8× size, gray color. No fanfare. |

**Rules:**
- NO card flip (no `reveal` phase, no cardback)
- NO screen shake
- NO particles
- NO `swoosh` or `impact` phases
- Card slides to discard at reduced opacity (0.7)
- Correct answer text shown for 1.5s in blue (existing behavior — no change needed)
- Haptic: `notifyWarning()` (existing — double gentle tap)
- Sound: existing `wrong-fizzle` cue
- Damage numbers: spawned but with reduced value label (e.g., append `×0.6` or just show the reduced number, per how AR-59.1 routes damage)

**New `CardAnimPhase` value:** `'charge-wrong'`
**Duration constant:** `CHARGE_WRONG_DURATION = 300`

The `fizzle` phase is now renamed conceptually: `fizzle` = card cannot play (insufficient AP / blocked). `charge-wrong` = card played at reduced power due to wrong answer. Keep `fizzle` as-is; `charge-wrong` is a new distinct state.

### Charge Build-Up Visuals (during fling gesture — AR-59.2's drag)

The drag-up gesture needs progressive visual feedback. This integrates with AR-59.2's drag thresholds.

| Drag Distance | CSS State | Visual |
|---------------|-----------|--------|
| 0–40px | (none — existing) | Card follows finger. Existing info overlay shows. |
| 40–60px | `.charge-building-1` | Golden `box-shadow: 0 0 8px rgba(255, 200, 0, 0.5)`. "CHARGE" text label at 50% opacity. |
| 60–80px | `.charge-building-2` | Glow intensifies: `box-shadow: 0 0 15px rgba(255, 200, 0, 0.8)`. "CHARGE" text at 100%. Card `scale: 1.1`. |
| 80px+ | `.charge-ready` | Green pulse: `box-shadow: 0 0 20px rgba(50, 255, 50, 0.9)`. "READY!" text. Card `scale: 1.15`. |

All CSS transitions: `transition: all 150ms ease-out` on the card element.

"CHARGE" / "READY!" text rendered as a sibling `<div class="charge-label">` inside the card wrapper (not a pseudo-element — easier to control with Svelte reactive classes).

The CSS classes are applied to the card element in `CardHand.svelte` based on `dragDeltaY` when the dragging card's index matches.

### Surge Turn Enhancement

During Surge turns (AR-59.4), Charge animations receive additional flair:

- Build-up glow is golden from the start (skip gray/neutral for the 0–40px range, start `.charge-building-1` immediately)
- Charge correct: `onParticleBurst(60, 0xFFD700)` instead of 35 (golden shower)
- Screen shake: `cameras.main.shake(120, 0.005)` instead of `(100, 0.004)`
- Green flash intensity: 0.3 instead of 0.2
- Card leaves a brief golden trail: CSS `filter: drop-shadow(0 0 6px gold)` decaying via keyframe on the impact phase

The Surge state must be passed as a prop or readable store to `CardCombatOverlay.svelte` — check AR-59.4 for how Surge state is exposed. If not yet implemented, add a `isSurgeTurn?: boolean` prop to the overlay for future AR-59.4 wiring.
