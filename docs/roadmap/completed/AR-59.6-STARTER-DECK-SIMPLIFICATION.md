# AR-59.6 — Starter Deck Simplification

## Overview

**Goal**: Simplify the starter deck from a variable 15–18 card pool (mixed mechanics, archetype-biased) to exactly 10 fixed cards: 5 Strike, 4 Block, 1 Surge. Remove archetype selection and starter relic selection from the run start flow entirely.

**Dependencies**: None — can be done independently of AR-59.1 (Quick Play / Charge mechanics).

**Estimated Complexity**: Medium — touches run start flow, deck builder, two UI screens, balance constants, and the experiment system.

---

## Design Decisions

- **D12**: Cut retreat-or-delve — run flow is simplified across the board
- **D25**: Onboarding redesign — no archetype selection, no forced Story Mode selection screen
- Starter deck is "deliberately boring" — interesting mechanics come from card rewards only
- 10 cards = cycle every 2 turns (draw 5). Each reward card = 10% deck change
- 3 AP per turn, draw 5. Charging a Strike costs 2 AP, leaving 1 AP. Resource tension from turn 1
- Surge (0 AP, draw 2) is the single interesting starter card. Charging Surge costs 1 AP

## Current State (as of 2026-03-16)

**Starter deck is NOT fixed.** The current path:

1. `gameFlowController.ts → onDomainsSelected()` → if runs ≥ 3, shows `archetypeSelection` screen; otherwise auto-assigns `'balanced'`

### Step 2 — Replace `buildStarterDeckFromRunPool` + `simplifyStarterMechanics` in `encounterBridge.ts`

**File**: `src/services/encounterBridge.ts`

Replace the two functions `simplifyStarterMechanics` (lines 154–201) and `buildStarterDeckFromRunPool` (lines 203–240) with a single new function `buildFixedStarterDeck`:

```ts
/**
 * Builds the fixed 10-card starter deck: 5 Strike, 4 Block, 1 Surge.
 * Cards are drawn from the run pool so they carry real fact IDs and domains.
 * Mechanic slots are filled strictly to STARTER_DECK_COMPOSITION ratios.
 */
function buildFixedStarterDeck(runPool: Card[]): Card[] {
  const result: Card[] = [];
  for (const { mechanicId, count } of STARTER_DECK_COMPOSITION) {
    const m = MECHANIC_BY_ID[mechanicId];
    if (!m) continue;
    const candidates = runPool.filter(c => !result.some(r => r.id === c.id));
    const picked = candidates.slice(0, count);
    for (const card of picked) {
      result.push({
        ...card,
        mechanicId: m.id,
        mechanicName: m.name,
        baseEffectValue: m.baseValue,
        originalBaseEffectValue: m.baseValue,
        apCost: m.apCost,
      });
    }
  }
  return result;
}
```

Update the call site (currently line 386–389):
```ts
// Before:
const starterDeck = simplifyStarterMechanics(
  buildStarterDeckFromRunPool(activeRunPool, run.starterDeckSize)
);
// After:
const starterDeck = buildFixedStarterDeck(activeRunPool);
```

Import `STARTER_DECK_COMPOSITION` from `'../data/balance'` (add to existing import).

**Acceptance**: Starter deck built from pool has exactly 10 cards matching the composition. `starterDeckSize` on `RunState` is no longer read by this path.
