# AR-59.18 — Surge Visual System

**Status:** Pending
**Depends on:** AR-59.4 (Knowledge Surge — must be complete; `isSurgeTurn` state and timing must be wired), AR-59.2 (CHARGE! button must exist for styling), AR-59.16 (Charge animations should coordinate with Surge enhancements)
**Blocks:** Nothing downstream

## Overview

Advanced visual effects for Knowledge Surge turns. AR-59.4 handles the core Surge LOGIC (timing, AP cost, state). This AR handles the VISUAL POLISH — the particles, the atmosphere shifts, the coordinated animations that make Surge FEEL like a power moment.

**Scope:** Phaser-side particle effects, ambient overlays, and coordinated multi-layer visuals that fire on Surge turn start and persist for the duration of the Surge turn, then transition out cleanly at turn end.

**Existing VFX infrastructure to build on:**
- `src/game/systems/CombatParticleSystem.ts` — multi-emitter system with `burstImpact`, `startAmbient`, `stopAmbient`, `comboMilestone`, etc. Has device-tier budgets (`particleBudget`, `ambientBudget`) and `reduceMotion` gating. Uses procedural textures (`vfx_square_4`, `vfx_circle_6`, `vfx_diamond_4`).
- `src/game/systems/CombatAtmosphereSystem.ts` — manages fog overlay and per-floor-theme ambient particles. Depth 997 for ambient emitters. Device tier budgets: low=10, mid=20, flagship=50.
- `src/game/managers/AudioManager.ts` — `playHazardSfx(key)` method for SFX. Keys must be preloaded in `BIOME_AUDIO`. Surge sound hook should call the Phaser scene's sound system directly.
- `src/ui/components/CardCombatOverlay.svelte` — has `synergy-flash` pattern (fade-in overlay div) and `pulse-glow` keyframe animations. `pulse-glow` keyframe already exists for `end-turn-pulse` button. Good reference for CSS overlay patterns.
- `src/ui/components/CardHand.svelte` — renders cards in hand. No current surge-related classes.

## Surge Visual Layers

### Layer 1: Screen Edge Golden Pulse (Announce — 150ms)
Screen edges pulse warm golden/amber once at Surge turn start. Does NOT persist.

**Implementation:** Full-screen `position: fixed` overlay div in `CardCombatOverlay.svelte` with `pointer-events: none`. On Surge turn start, add `surge-announce` CSS class that fires a one-shot `@keyframes surgeEdgePulse` animation (150ms). Remove class after animation completes.

```css
@keyframes surgeEdgePulse {
  0%   { box-shadow: inset 0 0 0px rgba(255, 193, 37, 0); }
  50%  { box-shadow: inset 0 0 60px rgba(255, 193, 37, 0.35); }
  100% { box-shadow: inset 0 0 0px rgba(255, 193, 37, 0); }
}
```

### Layer 2: AP Counter Transform (Announce → Persist)
- At Surge announce: AP counter area shows "SURGE" text overlay for 300ms (golden, bold)
- After 300ms: AP area transitions to show ⚡ lightning bolt icon + remaining AP number for the full duration of the Surge turn
- At Surge end: AP display returns to normal gem icons / number

### Layer 3: Card Golden Shimmer (Persist — full turn)
All cards in hand get a pulsing golden edge glow via `card-surge-glow` CSS class on each card wrapper element. Cards that are also chain-matched receive BOTH the chain color tint AND the golden surge glow (they stack, not override).

```css
.card-surge-glow {
  animation: surgeGlowPulse 2s ease-in-out infinite;
}
@keyframes surgeGlowPulse {
  0%, 100% { box-shadow: 0 0 8px rgba(255, 193, 37, 0.3); }
  50%       { box-shadow: 0 0 12px rgba(255, 193, 37, 0.6); }
}
```

When Surge ends, remove class with CSS transition (opacity 0 over 300ms before class removal).

### Layer 4: Golden Particle Overlay (Persist — Phaser canvas)
Subtle golden particles drifting upward on the combat background during Surge. Rendered BEHIND the enemy sprite but IN FRONT of the background (depth between atmosphere depth 997 and emitter depth 998).

**Primary:** New `startSurgeParticles()` / `stopSurgeParticles()` methods on `CombatParticleSystem`:
- Texture: `vfx_circle_6` tinted gold `0xFFD700`
- Spawn: continuous emitter, frequency 300ms (1 particle per 300ms)
- Velocity: y from -10 to -25 (upward), x drift ±5
- Alpha: fade from 0.4 to 0 over lifespan
- Lifespan: 2000ms
- Depth: 997.5 (between atmosphere at 997 and other effects at 998)
- Max particles: respects device-tier surge budget (see Device Adaptation)
- `stopSurgeParticles()` stops spawning only — existing particles continue their natural fade

**Fallback (if Phaser canvas not accessible from Svelte):** CSS particle overlay in `CardCombatOverlay.svelte` — `position: absolute` div with multiple small animated golden dots using CSS keyframe drift animations.

### Layer 5: CHARGE Button Enhancement (Persist)
During Surge, the CHARGE! button receives `charge-btn-surge` CSS class:

```css
.charge-btn-surge {
  background: linear-gradient(135deg, #FFD700, #FFA500) !important;
  animation: surgeBtnPulse 1.5s ease-in-out infinite;
  color: white;
  text-shadow: 0 1px 3px rgba(0,0,0,0.3);
}
@keyframes surgeBtnPulse {
  0%, 100% { transform: scale(1.0); }
  50%       { transform: scale(1.03); }
}
```

The "+0 AP" cost text is rendered in gold color instead of normal. This makes the "free Charge" affordance immediately visible.

### Layer 6: Sound Hook (Announce)
At Surge announce, call the Phaser sound system with key `'surge_announce'`. Sound file does not need to exist yet — the hook must be wired. If the key is not loaded, the call should fail silently (check `scene.cache.audio.has('surge_announce')` before playing).

Add `// TODO: surge_announce — low bass thrum, ~300ms, power-hum feel` comment alongside the hook.

## Surge End Transition (300ms)

1. Golden particle spawning stops immediately (`stopSurgeParticles()`)
2. Existing particles continue their natural lifespan (fade out over remaining ~2s)
3. Card golden glow fades via CSS transition (opacity 0 over 300ms, then remove class)
4. Brief "dim pulse" — combat container opacity dips to 0.95 for 100ms then returns to 1.0 (CSS transition or WAAPI). This "exhale" marks the transition back to normal.
5. AP display returns from ⚡ + number to normal gem icons
6. CHARGE button returns to normal styling (remove `charge-btn-surge` class)

### Step 4: CombatParticleSystem — Surge Particle Methods
**File:** `src/game/systems/CombatParticleSystem.ts`

Add two public methods:

```typescript
/**
 * Start continuous golden surge particle overlay.
 * Particles drift upward during Knowledge Surge turns.
 * Call once at Surge turn start.
 */
public startSurgeParticles(): void

/**
 * Stop surge particle emission.
 * Existing particles continue their natural lifespan and fade out.
 * Call at Surge turn end.
 */
public stopSurgeParticles(): void
```

Implementation details:
- New private member: `private surgeEmitter: Phaser.GameObjects.Particles.ParticleEmitter | null = null`
- `startSurgeParticles()`: guard with `if (this.reduceMotion) return`. Determine surge budget from device tier (see Device Adaptation). Create continuous emitter using `vfx_circle_6` tinted `0xFFD700`. Set depth to 997.5 (or nearest valid integer — Phaser depth is integer, so use 997 with explicit creation-order after atmosphere emitter). Frequency 300ms. Velocity y: `{min: -25, max: -10}`, x: `{min: -5, max: 5}`. Alpha `{start: 0.4, end: 0}`. Lifespan 2000. Max particles = surge budget.
- `stopSurgeParticles()`: call `surgeEmitter.stop()` (stops spawning), then schedule destroy after 2100ms to let existing particles complete. Set `surgeEmitter = null` after destroy.
- Update `destroy()` to also clean up `surgeEmitter` if set.

**Acceptance criterion:** `startSurgeParticles()` creates upward-drifting gold particles; `stopSurgeParticles()` stops spawning but lets existing particles fade. TypeCheck passes.

### Step 9: Reduced Motion Support
**All modified files**

When `prefers-reduced-motion: reduce` is active OR `card:reduceMotionMode` localStorage is `true`:

| Effect | Reduced Motion Behavior |
|--------|------------------------|
| Screen edge pulse (Layer 1) | Skip entirely — no overlay animation |
| AP counter "SURGE" text flash | Skip the 300ms text flash; go directly to ⚡ icon |
| Card golden glow pulse | Static golden border only — no pulsing animation (`box-shadow: 0 0 8px rgba(255, 193, 37, 0.5)`, no `animation` property) |
| Phaser particles (Layer 4) | Already gated by `CombatParticleSystem.reduceMotion` guard — skip automatically |
| CHARGE button pulse | Static golden background only — no scale animation |
| Surge end dim pulse | Skip entirely |

The static color indicators (golden border, ⚡ icon, golden button) MUST still appear even with reduced motion — they convey game state, not just decoration.

In Svelte components, check reduced motion with:
```typescript
const reduceMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches
  || JSON.parse(localStorage.getItem('card:reduceMotionMode') ?? 'false') === true
```

Apply `surge-reduce-motion` CSS class instead of animated classes when `reduceMotion` is true.

## Notes for Implementer

- The `CombatParticleSystem` uses integer depth values. Surge emitters at depth `997` will render above the atmosphere emitter at `997` if created after it (Phaser z-order within same depth level is creation order). No fractional depths needed.
- The `synergy-flash` pattern in `CardCombatOverlay.svelte` (lines ~113-116, ~1294-1297, ~1899-1910) is a good reference for the one-shot overlay pattern used by the screen edge pulse.
- The `pulse-glow` keyframe already defined in `CardCombatOverlay.svelte` (lines ~1696-1699) demonstrates the existing pulsing box-shadow animation pattern.
- `CombatParticleSystem.startAmbient()` uses `frequency: 500 / emitRate` for continuous emission rate — use same pattern for Surge emitter with `frequency: 300` (1 particle per 300ms).
- Do NOT modify `CombatAtmosphereSystem.ts` unless implementing the optional atmosphere gold tint shift. The Surge particles layer cleanly on top without changes.
- Update `docs/GAME_DESIGN.md` section 5.6 (VFX Systems) to document the Surge Visual System as a new subsystem alongside Combat Atmosphere and Status Effect Visual systems.
