# AR-59.17 — Knowledge Chain Visual System

## Overview

Visual indicators for the Knowledge Chain system. Cards in hand display colored tints by `categoryL2`, matching cards pulse in sync, and chain plays show connecting lines between chained cards. Milestone chain lengths (3+) trigger escalating celebration effects.

**Dependencies:** AR-59.3 (Knowledge Chain System) — chain state must exist to drive the visuals. The chain system is responsible for tracking which cards share a `categoryL2` and detecting when a chain is formed on play. This doc covers only the visual layer.

**Estimated complexity:** Medium. Primarily CSS animations and a small TypeScript service. No new game state.

---

## Context: Where `categoryL2` Lives

`categoryL2` is a field on the `Fact` object (see `src/data/types.ts`), NOT on the `Card` interface (`src/data/card-types.ts`). The `Card` interface has a `factId` that links back to the source fact.

When implementing, the chain visuals service needs to receive `categoryL2` values alongside or derived from the card's `factId`. The chain system (AR-59.3) should expose a per-card `categoryL2` lookup or store it on the card at draw time. The visual system should NOT query the fact DB directly at render time.

**Implementation assumption:** By the time AR-59.17 is implemented, each `Card` in the hand will have a `factCategoryL2?: string` field populated at draw time by the chain system. If that field is not yet added, Step 0 below covers it.

### Step 2: Card Frame Edge Tint

Modify `src/ui/components/CardHand.svelte`.

Each card in the hand shows a 2–3px colored stripe on its **left frame edge**. The color is derived from the card's `factCategoryL2` via `getChainColor()`.

**Implementation details:**

- Import `getChainColor` from `src/services/chainVisuals.ts`
- In the card's template markup, add an absolutely-positioned `<div>` inside each card container:
  ```html
  <div
    class="chain-edge"
    style="background: {getChainColor(card.factCategoryL2)};"
  ></div>
  ```
- CSS for `.chain-edge`:
  ```css
  .chain-edge {
    position: absolute;
    left: 0;
    top: 4px;
    bottom: 4px;
    width: 3px;
    border-radius: 2px 0 0 2px;
    opacity: 0.85;
    pointer-events: none;
    z-index: 2;
  }
  ```
- The stripe must be positioned so it is visible in the fan layout (the left edge is exposed even when cards overlap). The existing card container already uses `position: relative`.
- If `factCategoryL2` is undefined, `getChainColor` returns Gray — still render the stripe (indicates "no category chain possible").

**Acceptance criterion:** Every card in hand shows a 2–3px colored left-edge stripe. Color is consistent for the same `categoryL2` across different cards and game sessions. Stripe does not interfere with existing card interaction (pointer events disabled).

### Step 3: Matching Card Pulse Animation

Modify `src/ui/components/CardHand.svelte`.

When 2+ cards in hand share the same `categoryL2`, their colored edge stripes pulse in sync.

**Implementation details:**

- Derive a `pulsingCategoryL2s: Set<string>` from the current `cards` array:
  ```ts
  const pulsingCategoryL2s = $derived.by(() => {
    const counts = new Map<string, number>()
    for (const card of cards) {
      if (card.factCategoryL2) counts.set(card.factCategoryL2, (counts.get(card.factCategoryL2) ?? 0) + 1)
    }
    return new Set([...counts.entries()].filter(([, n]) => n >= 2).map(([k]) => k))
  })
  ```
- Add `chain-pulse` class to `.chain-edge` when `pulsingCategoryL2s.has(card.factCategoryL2 ?? '')`:
  ```html
  <div
    class="chain-edge"
    class:chain-pulse={pulsingCategoryL2s.has(card.factCategoryL2 ?? '')}
    style="background: {getChainColor(card.factCategoryL2)};"
  ></div>
  ```
- CSS animation:
  ```css
  @keyframes chain-pulse-anim {
    0%, 100% { opacity: 0.85; }
    50%       { opacity: 0.35; }
  }
  .chain-edge.chain-pulse {
    animation: chain-pulse-anim 1.5s ease-in-out infinite;
  }
  ```
- All cards in the same group share identical animation timing (they start at the same phase because the keyframes reset on class add, and all cards enter at the same hand draw moment).

**Acceptance criterion:** When exactly 1 card has a given `categoryL2`, its edge does NOT pulse. When 2+ cards share `categoryL2`, all of their edges pulse together. Pulse does not occur on categories with only 1 representative.

### Step 4: Chain Play Connecting Line

Modify `src/ui/components/CardHand.svelte`.

When a chain is formed (the 2nd+ card played with the same `categoryL2` in sequence), a thin glowing line briefly connects the just-played card to the previous chain card in the hand area.

**Props needed from parent:** The parent (`CardCombatOverlay.svelte`) must pass down chain state — specifically the card IDs of the current chain sequence and whether a chain event just fired. Add these to `CardHand`'s `Props` interface:

```ts
chainSequence?: string[]        // cardIds in current chain (from chain system)
chainJustFired?: boolean        // true for ~300ms after a chain play
```

**Implementation details:**

- Add an SVG overlay element above the card hand area (using `position: absolute`, `pointer-events: none`, `overflow: visible`)
- When `chainJustFired` is true and `chainSequence` has 2+ entries, draw SVG `<line>` elements between the screen positions of consecutive chain cards
- Card positions are known from the existing fan layout calculation (`getXOffset`, `getArcOffset`, `getRotation`). Store DOM refs per card index to get actual bounding rects if needed, or calculate from the same layout formulas
- Line style:
  - `stroke`: chain color of the shared `categoryL2`
  - `stroke-width`: 2
  - CSS: `filter: drop-shadow(0 0 4px <chain-color>)` for glow
  - Opacity animates 1 → 0 over 300ms after trigger (CSS transition on opacity)
- For 3+ chains: draw lines card[0]→card[1], card[1]→card[2], etc.

**Acceptance criterion:** On the 2nd card play in a chain, a glowing colored line briefly appears connecting the two most-recently-chained card positions and fades within 300ms. For 3-chains, two connected line segments appear. Line does not block card interaction.

### Step 5: Chain Milestone Effects (3+)

Modify `src/ui/components/CardCombatOverlay.svelte`.

Escalating effects at chain milestones. The chain length is passed down from the chain system (AR-59.3).

**3-Chain:**
- All currently-pulsing chain-edge stripes briefly switch to a bright glow (box-shadow: 0 0 8px 3px `<color>`)
- Short particle trail on the played card (3–5 small dots that scatter outward, CSS keyframe, 400ms total)
- Subtle audio cue (reuse existing audio system: `audioManager.playSound('chain_3')` — register this sound key)

**4-Chain:**
- Screen edge pulse: a thin gold border (`box-shadow: inset 0 0 0 3px #ffd700`) appears on the game container div, fades out over 200ms

**5-Chain:**
- Screen shake: CSS `@keyframes screen-shake` applied to the main combat container (`translateX` oscillation ±4px, 4 cycles, 300ms total)
- "KNOWLEDGE CHAIN!" text overlay: centered fixed text, bold white with gold text-shadow, font-size scaled to 7% of container height, fades in over 100ms, holds 400ms, fades out over 300ms (total 800ms)
- Particle burst: 8–12 dots radiate from the center of the screen, colored with the chain color

**Implementation details for `CardCombatOverlay.svelte`:**

- Accept a `chainLength: number` prop (0 = no chain, 1 = first in sequence, 2+ = chain active)
- Use `$effect` to watch `chainLength` and trigger milestone effects
- All milestone elements are rendered conditionally via `$state` flags (`showChain3Glow`, `showChain4EdgePulse`, `showChain5Celebration`) that are set to true and then auto-reset after their animation duration via `setTimeout`
- The "KNOWLEDGE CHAIN!" text and particle burst live in a `<div class="chain-celebration-overlay">` inside the combat container with `position: absolute; inset: 0; pointer-events: none; z-index: 100`

**Acceptance criterion:**
- 3-chain: bright glow on chain edge stripes, particle trail from played card position
- 4-chain: gold screen border flash (200ms)
- 5-chain: screen shake + "KNOWLEDGE CHAIN!" text visible for ~800ms then fades
- Effects do not block card play or quiz interaction
- All effects are purely cosmetic — no game state side effects
