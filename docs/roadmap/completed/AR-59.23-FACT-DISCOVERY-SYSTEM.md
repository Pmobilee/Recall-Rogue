## Overview

The Fact Discovery System solves the Charge cold-start problem: paying +1 AP to Charge a card you've never seen before is a blind guess that will often fail. The original design (discovery overlay + cardHint hints on cards) was scrapped because it created **card avoidance behavior** — players would see "this card has no hint = unknown fact" and deliberately skip it.

**The new design: Free First Charge.** The first time a fact appears in a run, the player can Charge it for FREE (+0 AP surcharge). If wrong, no penalty (1.0× = same as Quick Play). If correct, full Charge bonus (2.5×/3.0×/3.5× per tier).

This gives players a free "try it and see" on every new fact. No risk, only upside. No visual markers, no hint text, no card avoidance.

**The ONLY visual indicator:** The CHARGE button displays **"FREE"** (instead of "+1 AP") when a fact hasn't been Charged yet this run.

**The natural run arc this creates:**
- **Act 1:** Most facts are new → lots of free Charges → players learn which facts they know
- **Act 2:** Mix of free and paid Charges → players Charge facts they learned in Act 1
- **Act 3:** Mostly paid Charges → players are experts on their discovered facts

Design spec source: `docs/RESEARCH/fact-discovery-system.md`

### Step 2: Create `src/services/discoverySystem.ts`

**File:** `src/services/discoverySystem.ts` (NEW FILE)

```typescript
/**
 * Fact Discovery System — "Free First Charge"
 *
 * The first time a player Charges a fact in a run, it is FREE:
 * - AP surcharge = 0 (regardless of Surge state)
 * - Wrong answer = 1.0× multiplier (same as Quick Play — no penalty for trying)
 * - Correct answer = full Charge multiplier (2.5×/3.0×/3.5× per tier)
 *
 * After the first Charge, the fact is normal for the rest of the run:
 * - AP surcharge = +1
 * - Wrong answer = 0.7× multiplier
 *
 * Per-run, not per-encounter. Resets only on new run.
 */

/**
 * Returns true if this fact has NOT yet had its free first Charge used this run.
 * A free Charge is available for every fact once per run.
 */
export function isFirstChargeFree(
  factId: string,
  freeChargeIds: Set<string>,
): boolean {
  return !freeChargeIds.has(factId);
}

/**
 * Marks a fact's free first Charge as used. Call this after Charge resolution
 * (regardless of correct/wrong outcome).
 *
 * Mutates the freeChargeIds set in place.
 */
export function markFirstChargeUsed(
  factId: string,
  freeChargeIds: Set<string>,
): void {
  freeChargeIds.add(factId);
}

/**
 * Returns the damage multiplier for a wrong answer on a free first Charge.
 * Same as Quick Play — 1.0× — so there is no penalty for trying a new fact.
 */
export function getFirstChargeWrongMultiplier(): number {
  return 1.0;
}
```

- [ ] File created at `src/services/discoverySystem.ts`

- [ ] `markFirstChargeUsed()` adds factId to set

### Step 3: Wire into `turnManager.ts` (Charge play path)

**File:** `src/services/turnManager.ts`

In the Charge play resolution path (wherever wrong/correct multipliers are applied after quiz answer):

**Before quiz resolves:**
1. Check `isFirstChargeFree(card.factId, runState.firstChargeFreeFactIds)`
2. If `true`: set AP surcharge to 0 for this play (override the normal +1 AP deduction)

**After quiz resolves:**
- If wrong AND free Charge: apply `getFirstChargeWrongMultiplier()` (1.0×) instead of the normal wrong-Charge multiplier (0.7×)
- If correct AND free Charge: apply full Charge multiplier normally (no change — correct answer always gets the full bonus)
- After resolution (win or lose): call `markFirstChargeUsed(card.factId, runState.firstChargeFreeFactIds)`

**Tier 3 auto-Charge exception:** Tier 3 cards auto-Charge without quizzing the player. Auto-Charge does NOT consume the free Charge (the player didn't actually attempt an answer). Do not call `markFirstChargeUsed()` for auto-Charge resolutions.

**AP surcharge timing:** The AP is deducted when Charge is initiated (before the quiz). For free Charges, deduct 0 AP surcharge (base card AP cost still applies, only the +1 surcharge is waived). Find where the `+1 AP` Charge surcharge is deducted and gate it:

```typescript

const chargeSurcharge = isFreeChrg ? FIRST_CHARGE_FREE_AP_SURCHARGE : CHARGE_AP_SURCHARGE;
// deduct chargeSurcharge from currentAP
```

## Design Notes

### Why "FREE" Instead of Hint Text

The original design showed a `cardHint` (e.g., "Camel eyelids?") on discovered cards. This created a silent anti-pattern: players would see cards WITHOUT hints and consciously avoid Charging them. "No hint = unknown = risky" is a perfectly rational avoidance strategy — but it works against the educational goal of encouraging exploration.

The Free First Charge design removes this avoidance vector entirely. Players see no per-card visual state. The only signal is the CHARGE button label: "FREE" means try it, "+1 AP" means you've seen it before. The button is already the action point — changing the label there is natural and non-stigmatizing.

### Why 1.0× for Wrong Answer

A wrong first Charge costs 0 extra AP and deals 1.0× damage (same as Quick Play). This means the downside of trying a new Charge is literally zero — it's identical to just Quick Playing the card. Players have no reason NOT to free-Charge every new fact they see.

This is intentional: maximum exploration incentive. The game wants players to Charge everything at least once so they learn what they know and don't know. The penalty structure can tighten once a fact is "known" (i.e., after the free Charge is used).

### Run Arc Without Visual States

- **Act 1:** Almost every CHARGE button says "FREE" → players naturally Charge everything → discover what they know
- **Act 2:** Mix of "FREE" and "+1 AP" buttons → players selectively Charge facts they answered correctly in Act 1
- **Act 3:** Mostly "+1 AP" buttons → players are veterans of their fact pool, Charge with confidence

This recreates the exploration → mastery arc from the original design without any card-level visual states that could cause avoidance.

### Tier 3 Auto-Charge Preservation

Tier 3 cards auto-Charge without quizzing. If their free Charge were consumed, the player would arrive at future encounters and see "+1 AP" on a card they never consciously Charged. This is confusing and wastes the free Charge on an interaction the player didn't choose. Auto-Charges skip the free-Charge system entirely.
