# AR-59.7 â€” Boss Quiz Phases

## Overview
Add Boss Quiz Phases: combat pauses at HP thresholds for rapid-fire quiz rounds. This is a new combat mechanic unique to boss encounters that turns boss fights into a culminating test of knowledge. Even Quick-Play-only players must answer questions during quiz phases â€” they are the mandatory knowledge moments of the run.

**Dependencies:**
- AR-59.5 (3-Act Run Structure) â€” bosses assigned to acts; Archivist is Act 2, Curator is Act 3
- AR-59.1 (Charge System) â€” provides quiz infrastructure; Charged plays supply categoryL2 accuracy data
- AR-59.13 (Enemy Roster) â€” Archivist and Curator enemy definitions (both already exist in `src/data/enemies.ts`)

**Complexity:** High â€” new service, new TurnState fields, new UI mode, combat flow interruption

---

## Design Decisions

- **D19 â€” Weakest Domain:** For Curator phase 1, "weakest domain" = the `categoryL2` label with the lowest `correct/total` ratio across all facts answered during the current run. Tracked in `RunState`, updated after every card play (correct or incorrect). If no facts have been answered yet (impossible by Act 3 but defensively handled), fall back to the `categoryL2` with the most facts in the current run pool.
- **Boss quiz phases are mandatory:** They interrupt combat fully. Cards are hidden, normal play is suspended. No way to skip or defer.
- **No double-trigger:** Each phase fires at most once per encounter. Tracked by index in `TurnState`.
- **Combat resumes at the same turn:** Player does NOT lose their turn. After the phase, play continues with the same AP, hand, and combo state that existed when the phase triggered.

---

## Boss Quiz Phase Specification

### The Archivist (`the_archivist`) â€” Act 2 Boss â€” 1 Quiz Phase
- **Trigger:** At 50% HP (HP drops to or below 50% of max)
- **Questions:** 5, drawn from the run's current fact pool (standard distribution)
- **Timer:** Standard floor-based timer (same as card play)
- **Per correct answer:** Boss loses 10% of its remaining HP + player gains a random buff (one of: Empower status +2 strength for 2 turns, +2 block, draw 1 card)
- **Per wrong answer:** Boss gains +3 Strength (stacks, applied as `strength` status effect, value 3, turns 99)
- **After phase:** Combat resumes. Accumulated buffs/debuffs persist.

### The Curator (`the_curator`) â€” Act 3 Final Boss â€” 2 Quiz Phases

**Phase 1 â€” at 66% HP:**
- **Questions:** 5, sourced from the player's weakest `categoryL2` (lowest run accuracy)
- **Timer:** Standard floor-based timer
- **Per correct answer:** Boss loses 10% of its remaining HP
- **Per wrong answer:** Boss gains +3 Strength

**Phase 2 â€” at 33% HP â€” RAPID FIRE:**
- **Questions:** 8, drawn from the run's fact pool
- **Timer:** Hard override â€” 4 seconds per question regardless of floor or slow reader setting
- **Per correct answer:** Boss takes 5 direct damage immediately (visible HP bar tick-down during phase)
- **Per wrong answer:** Boss heals 5 HP immediately (visible HP bar tick-up during phase)
- **Timer bar color:** Red (urgency signal)
- **Transition between questions:** 200ms (vs standard 300ms)
- **This is the climactic test of everything learned during the run**

## UX Flow (per phase)

1. Enemy HP crosses threshold during `playCardAction` â†’ quiz phase check fires
2. Combat state transitions to `'quizPhase'`; turn flow frozen (no card plays, no enemy turn)
3. Screen dims slightly (overlay `opacity: 0.3` behind UI)
4. "QUIZ PHASE" text flashes center screen (dramatic zoom-in animation, 500ms, `QUIZ_PHASE_ANNOUNCE_DURATION_MS`)
5. Cards in hand slide down off-screen
6. Questions appear one at a time, center screen:
   - Question text at top
   - 3â€“4 answer buttons (full width, `56dp` height)
   - Timer bar (standard or overridden); red in rapid-fire mode
   - After each answer: 300ms feedback flash (green = correct, red = wrong), auto-advance
7. After all questions: results summary card â€” "X / Y Correct" â€” displayed for 2 seconds (`QUIZ_PHASE_RESULTS_DURATION_MS`)
   - >60% correct: green theme, "Boss takes damage!" text
   - <40% correct: red theme, "Boss powers up!" text
   - 40â€“60%: neutral
8. Boss reaction animation:
   - >60% correct â†’ knockback tween, red flash, HP bar drains visibly, stun 0.5s
   - <40% correct â†’ scale to 1.1x, red glow, Strength icon appears, threatening animation
   - 40â€“60% â†’ boss returns to idle
9. Rewards/penalties applied to boss and player state
10. Cards slide back up into hand; `'quizPhase'` state clears; combat resumes (same turn, same AP)
11. Total interruption: ~15â€“30 seconds depending on question count

### Step 2 â€” Create `src/services/bossQuizPhase.ts`

New service file. Pure logic, no DOM or Phaser imports.

**Exports:**

```ts
/**
 * Check whether a boss quiz phase should trigger given the boss's current HP.
 * Returns the matching phase config if a new threshold was crossed, or null.
 * Checks against `alreadyTriggeredPhaseIndices` to prevent re-triggering.
 */
export function checkQuizPhaseThreshold(
  bossId: string,
  currentHpPercent: number,
  alreadyTriggeredPhaseIndices: number[],
): { config: BossQuizPhaseConfig; phaseIndex: number } | null

/**
 * Select questions for a quiz phase.
 * For useWeakestDomain=true: finds the categoryL2 with lowest accuracy in runState,
 * then draws `config.questionCount` facts from that categoryL2 group.
 * For standard phases: draws from the run's general fact pool.
 * Returns an array of QuizQuestion objects ready for display.
 */
export function generateQuizPhaseQuestions(
  config: BossQuizPhaseConfig,
  runState: RunState,
  factPool: CardRunState['library'],
): QuizQuestion[]

/**
 * Calculate and return the outcome of a completed quiz phase.
 * Does NOT mutate enemy or player state â€” returns a structured outcome
 * for the combat system to apply.
 */
export function resolveQuizPhaseResults(
  config: BossQuizPhaseConfig,
  results: { correct: number; wrong: number },
  enemyCurrentHP: number,
): QuizPhaseOutcome
```

**Types to define in this file:**
```ts
export interface QuizQuestion {
  factId: string;
  question: string;
  answers: string[];    // shuffled, includes correct answer
  correctAnswer: string;
  categoryL2?: string;
}

export interface QuizPhaseOutcome {
  bossDamage: number;           // total HP to remove from boss
  bossHeal: number;             // total HP to restore to boss (rapid fire wrong answers)
  bossStrengthGain: number;     // total Strength stacks to add to boss
  playerRandomBuffs: number;    // number of random buffs to grant player (Archivist)
  correctCount: number;
  wrongCount: number;
  totalCount: number;
  theme: 'positive' | 'negative' | 'neutral'; // for UI results screen
}
```

**Weakest domain logic in `generateQuizPhaseQuestions`:**
1. Iterate over `runState.categoryL2Accuracy` (added in Step 5)
2. For each `categoryL2`, compute `correct / total` â€” minimum 0 if `total === 0`
3. Select the `categoryL2` with the lowest ratio
4. Filter `factPool` to only facts where `fact.categoryL2 === weakestCategoryL2`
5. If fewer than `config.questionCount` facts in that group, take all available + fill remainder from general pool
6. Shuffle and slice to `config.questionCount`

if (effect.damageDealt > 0 && !effect.enemyDefeated && enemy.template.id in BOSS_QUIZ_PHASES) {
  const hpPercent = enemy.currentHP / enemy.maxHP;

 * Resume normal combat after a quiz phase has concluded.
 * Applies the outcome to enemy and player state, then transitions
 * the phase back to 'player_action'.

  turnState: TurnState,
  outcome: QuizPhaseOutcome,

Implementation of `resumeAfterQuizPhase`:

- Set `turnState.phase = 'player_action'`
- Set `turnState.pendingQuizPhaseIndex = null`

### Step 8 â€” Rapid Fire Real-Time HP Updates

In rapid-fire mode, damage and healing are applied per-answer rather than in batch at phase end.

In `CardCombatOverlay.svelte` `questioning` stage handler for rapid-fire:
```ts
if (isRapidFireMode) {
  if (answeredCorrectly) {
    // Apply 5 direct damage immediately
    enemy.currentHP = Math.max(0, enemy.currentHP - QUIZ_PHASE_RAPID_FIRE_VALUE);
    if (enemy.currentHP <= 0) {
      // Boss defeated mid-phase â€” end quiz phase, trigger victory
      endQuizPhaseEarly('victory');
      return;
    }
  } else {
    // Boss heals 5 HP immediately
    enemy.currentHP = Math.min(enemy.maxHP, enemy.currentHP + QUIZ_PHASE_RAPID_FIRE_VALUE);
  }
  // HP bar updates reactively via existing store binding
}
```

The reactive HP bar in the combat UI already reads `enemy.currentHP` and will update in real time.

At phase end for rapid-fire, `resolveQuizPhaseResults` returns `bossDamage: 0` and `bossHeal: 0` (since effects were already applied per-answer); `resumeAfterQuizPhase` still applies strength/buff outcomes.

**Acceptance:** In rapid-fire mode, boss HP bar visibly ticks up/down after each answer during the phase.
