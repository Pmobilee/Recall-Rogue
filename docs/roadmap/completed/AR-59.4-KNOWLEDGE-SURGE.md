## Overview

Every 3rd player turn is a **Knowledge Surge** turn where Charging costs +0 AP instead of +1. This creates a predictable combat rhythm — players know when burst windows are coming — and rewards knowledge-focused play at regular intervals.

**Turn sequence:** Normal → SURGE → Normal → Normal → SURGE → Normal → Normal → SURGE → ...

Surge is the "exciting moment" in combat rhythm, replacing the Knowledge Combo system's role as the primary dopamine spike.

---

## Design Decisions (locked — do not re-litigate)

| # | Decision |
|---|----------|
| D9 | First Surge on turn 2 (not turn 3). Short fights still get a Surge window. |
| D10 | All 3 quizzes on a Surge turn are free (Charging costs +0 AP, not just the first 2). If playtesting reveals this is overwhelming, fallback is "first 2 Charges free, 3rd costs +1 AP". |
| — | Surge REPLACES the Knowledge Combo system as the "exciting burst window" in combat. |
| — | Surge is purely AP-cost based. Charge multipliers (2.5×, 3.0×, 3.5×, 1.2×) do NOT change during Surge. |
| — | Tier 3 auto-Charge is already free (no AP surcharge). Surge has no additional effect on Tier 3 cards. |

---

## Surge Timing Formula

```
isSurgeTurn(n) = (n === 2) || (n > 2 && (n - 2) % 3 === 0)
```

| Turn | Is Surge? |
|------|-----------|
| 1 | No |
| 2 | **YES** (first Surge) |
| 3 | No |
| 4 | No |
| 5 | **YES** |
| 6 | No |
| 7 | No |
| 8 | **YES** |
| 11 | **YES** |
| 14 | **YES** |

Surge state resets per encounter — a new fight always starts at turn 1.

---

## Surge Announcement Sequence (0.5s total, non-interruptive)

| Phase | Duration | Effect |
|-------|----------|--------|
| Flash | 0.15s | Screen edges pulse warm golden/amber once |
| Label | 0.3s | AP counter area briefly shows "SURGE" text replacing the number |
| Sound | 0.3s | Low satisfying bass thrum (power-up hum) — use existing audio system |
| Persist | Full turn | Cards have golden shimmer edge, AP shows ⚡ icon, gold particle overlay |

The sequence is non-interruptive — the player can take actions during or immediately after the announcement. No blocking modal or forced wait.

---

## During a Surge Turn

- All cards in hand have a persistent golden shimmer edge (`card-surge-glow` CSS class)
- CHARGE! button shows "+0 AP" instead of "+1 AP" (and applies gold styling)
- Fling gesture threshold reduced to 60px (from 80px after AR-59.2) — Charging feels "easier"
  - The golden glow preview starts at 30px instead of 40px
- AP counter shows ⚡ lightning bolt icon next to the current value
- Combat background: subtle golden particle overlay (CSS keyframe animation)
- Card Charge cost tooltip / info display shows "0" instead of "+1"

---

## Surge End

When the Surge turn ends (player clicks End Turn or hand is exhausted):

1. Remove `surge-active` CSS class with a 0.3s CSS transition
2. Golden glow fades smoothly off all cards
3. Brief dim pulse (opacity dip to 0.95 and back, 200ms) signals return to normal
4. Normal card colors and CHARGE! button cost restore immediately

### Step 2 — Create `src/services/surgeSystem.ts`

New file. This replaces the `isSurgeTurn()` stub left by AR-59.1 in `turnManager.ts`.

```ts
/**
 * Knowledge Surge System (AR-59.4)
 *
 * Every 3rd player turn is a Surge turn where Charging costs +0 AP.
 * Surge turns: 2, 5, 8, 11, 14, ...
 * Resets per encounter (turnNumber resets to 1 on new encounter).
 */

import { SURGE_FIRST_TURN, SURGE_INTERVAL } from '../data/balance';

/**
 * Returns true if the given turn number is a Surge turn.
 *
 * Formula: turn 2 is always first Surge. Then every SURGE_INTERVAL turns after.
 * isSurgeTurn(1) = false
 * isSurgeTurn(2) = true
 * isSurgeTurn(5) = true
 * isSurgeTurn(8) = true
 */
export function isSurgeTurn(turnNumber: number): boolean {
  if (turnNumber === SURGE_FIRST_TURN) return true;
  if (turnNumber > SURGE_FIRST_TURN && (turnNumber - SURGE_FIRST_TURN) % SURGE_INTERVAL === 0) return true;
  return false;
}

/**
 * Returns the AP surcharge for Charge Play on this turn.
 * During a Surge turn: 0 (free Charging).
 * During normal turns: 1 (standard surcharge from AR-59.1).
 */
export function getSurgeChargeSurcharge(turnNumber: number): number {
  return isSurgeTurn(turnNumber) ? 0 : 1;
}
```

## Acceptance Criteria

- [ ] Turn 2 is a Surge turn (first Surge fires early for short fights)
- [ ] Surge recurs every 3 turns: 2, 5, 8, 11, 14...
- [ ] Charging costs +0 AP surcharge during Surge turns
- [ ] Charging still costs +1 AP surcharge on normal turns (regression check)
- [ ] CHARGE! button displays "+0 AP" and amber/gold styling during Surge
- [ ] Surge announcement plays at turn start: golden edge flash (0.15s) + "SURGE" AP label (0.3s)
- [ ] AP counter shows ⚡ icon during full Surge turn
- [ ] Cards glow golden (`card-surge-glow`) during Surge turn
- [ ] Fling gesture threshold reduced to 60px during Surge (vs 80px normal)
- [ ] Fling preview glow starts at 30px during Surge (vs 40px normal)
- [ ] Surge state resets on new encounter (new `startEncounter` → `isSurge = false`, turn 1)
- [ ] Surge end: golden glow fades with 0.3s CSS transition
- [ ] Tier 3 auto-Charge AP cost is unaffected by Surge (it was already 0 surcharge)
- [ ] AR-59.1 `isSurgeTurn` stub removed from `turnManager.ts`
- [ ] `npx vitest run` passes with 0 failures
- [ ] `npm run typecheck` passes clean
- [ ] `npm run build` succeeds
- [ ] `docs/GAME_DESIGN.md` updated with Surge system description
- [ ] `docs/ARCHITECTURE.md` updated with `surgeSystem.ts` and `TurnState.isSurge`
