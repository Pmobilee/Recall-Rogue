# AR-59.19 — Remove Deprecated Systems (v2 Spec Alignment)

## Overview

**Goal:** Archive or remove systems that the v2 overhaul has replaced, while preserving systems confirmed to carry forward. This is a cleanup pass that follows the v2 feature work (AR-59.1–59.18) — it removes the old rails so only v2 systems remain in the production build.

**Estimated complexity:** Medium — touches many files but each change is a deletion or flag-gate, not new logic.

**Approach:** Feature-flag or move to `src/_archived/` where safe. Do NOT hard-delete unless the system has zero save-data implications. Leave resolution stubs for anything that may appear in existing save files.

---

| System | Replaced By | Archive Strategy |
|--------|-------------|-----------------|
| Knowledge Combo (consecutive correct → multiplier up to 2.0×) | Knowledge Chains (AR-59.3, categoryL2-based) | Feature-flag `ENABLE_KNOWLEDGE_COMBO = false`; remove UI counter |

| Relic synergy system (`relicSynergyResolver.ts`) | Simpler direct relic interactions in v2 | Archive file; remove call sites |

**In `src/services/cardEffectResolver.ts`:**
- Find the combo multiplier application in the damage/effect stack
- Wrap in `if (ENABLE_KNOWLEDGE_COMBO) { ... }` — do not delete the branch

**In `src/services/turnManager.ts`:**
- Find combo counter increment/reset logic
- Wrap in `if (ENABLE_KNOWLEDGE_COMBO) { ... }`

**In `src/ui/stores/gameState.ts`:**
- The `comboCount` / `knowledgeCombo` state field should remain in the store type for save-data compat, but its value will never increment (resolver is gated)

**In `src/ui/components/CardCombatOverlay.svelte`:**
- Remove the combo counter display element from the template
- Remove any reactive `$:` / `$derived` bindings that drive combo UI

**`src/ui/components/ComboCounter.svelte`:**

- Remove any import of `ComboCounter` from its parent(s)

**`src/ui/utils/comboDisplay.ts`:**
- Archive: add `@deprecated AR-59.19` JSDoc at the top; remove all import sites

**Acceptance:**
- No combo counter is visible in the combat UI
- `ENABLE_KNOWLEDGE_COMBO = false` gates all combo logic
- Combo state field still exists in store type (save compat)

**In `src/data/balance.ts`:**
- Remove `FIZZLE_EFFECT_MULTIPLIER` (0.2) and any related constants
- Do not replace — the Charge wrong-answer multiplier lives in the AR-59.1 balance constants

**In `src/services/cardEffectResolver.ts`:**
- Remove the fizzle branch entirely (hard removal, not flagged)
- Wrong-answer resolution now falls through to the Charge wrong-answer path from AR-59.1

**Acceptance:**
- `FIZZLE_EFFECT_MULTIPLIER` constant does not exist in `balance.ts`
- No fizzle application in `cardEffectResolver.ts`
- Wrong answers in Charge mode correctly apply 0.6× or 0.7× (from AR-59.1)

### Step 8: Update Bounty Quests for Charge/Chain Mechanics

Bounty quest definitions reference quiz-based goals that no longer exist in v2. Reword them to reference v2 Charge and Chain mechanics.

**`src/services/bountyManager.ts`:**
- Find quest definitions that reference: "answer X questions correctly," "achieve X combo," "answer X in a row," "fast answer," "speed bonus"
- Replace with v2-compatible wording:
  - "Answer X questions" → "Charge X cards in one encounter"
  - "Achieve a combo of X" → "Complete a chain of length X"
  - "Answer X in a row correctly" → "Charge X cards without a wrong answer"
  - "Get a speed bonus" → "Charge a card with a speed bonus" (requires Quicksilver Quill; gate this quest on relic ownership, or replace with a different quest)
  - "Correct answers" counts → "Charged cards" counts
- Remove any quests that reference archived systems (combo streaks, fizzle recoveries)

**Acceptance:**
- No bounty quest text references removed systems
- All quests are achievable via v2 mechanics
- Quests appear correctly in the Journal screen

### Step 9: Retune Canary System Thresholds

The Canary system adapts difficulty based on player accuracy. With v2's lower quiz frequency (45–70 Charge quizzes per run vs. 150+ in v1), the original thresholds trigger too slowly.

**`src/services/canaryService.ts`:**
- Find the assist-tier thresholds (wrong-answer counts and correct-streak counts)
- Update to v2 values (lower counts, more responsive to signal from fewer quizzes):
  - Deep Assist tier: trigger after **3+ wrong Charges** in the current encounter (down from 5+ wrong answers)
  - Assist tier: trigger after **2+ wrong Charges** in the current encounter (down from 3+)
  - Challenge tier: trigger after **4+ correct Charges** in a row (down from 5+ correct streak)
- Document the rationale in a comment: fewer quizzes per run means each data point carries more weight

**`src/data/balance.ts`:**
- Update corresponding Canary threshold constants to match the new values above

**Acceptance:**
- Canary thresholds reflect the new values
- Deep Assist activates after 3 wrong Charges in an encounter
- Challenge mode activates after a 4-correct-Charge streak
- Constants in `balance.ts` match values in `canaryService.ts`
