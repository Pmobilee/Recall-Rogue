### 4.2 Narration content picks facts player never saw + must work across all deck types

Example screenshot: a narrative sentence quotes almost verbatim the text of a quiz question the player **never actually encountered** in the run. The narration should only source from facts the player has actually seen this run, AND should not copy entire question text.

**Decisions:**
- Source pool = facts the player has **either answered correctly OR seen-and-missed** this run (both count as "encountered").
- Use a **lightweight entity/noun extractor** (regex + existing `DeckFact` metadata like `category`, `tags`, `entities` fields) — do NOT add a heavy NLP library. Minimal and deterministic.
- Narration should reference entities vaguely/evocatively — e.g. `"if only they knew about Catherine the Great…"` — not copy the whole question.

**CRITICAL — must work for ALL deck types, think hard about this:**

The narration system currently assumes an entity-noun model (good for history/science/geography). It will fail for other deck categories:

- **Math decks** — there are no "entities", only concepts (integrals, derivatives, Pythagorean theorem). Narration should reference concepts generically: *"the shape of a problem yet unsolved"*, *"numbers that refuse to resolve"*.
- **Foreign languages — vocabulary** — the "fact" is a word-translation pair. Narration should reference **the English translation**, not the foreign word: *"words for 'courage' whispered in a tongue they once knew"*.
- **Foreign languages — grammar** — the "fact" is a grammar pattern. Narration should reference the **pattern or its function** abstractly: *"the shape of conditional regret"*, *"verbs that twist when touched by negation"*.
- **Japanese specifically** — verbs, kanji, grammar patterns each need their own narration style. Probably one template family per subject-type.

**Implementation:** narration template selection must branch on deck category / fact type. Define a per-fact-type narration adapter — each adapter knows how to turn a fact into a narration reference. Register adapters for at minimum: `history`, `science`, `geography`, `mythology`, `math`, `vocab_foreign`, `grammar_foreign`, `kanji`.

**This decision must be captured in:**
- `docs/mechanics/` narration docs
- `.claude/skills/narration/` skill definition (add the adapter model)
- `docs/GAME_DESIGN.md` narration section

## Chapter 7 — Chain System & Charge AP Cost

**Owner agent:** game-logic (rules) + ui-agent (indicators)
**Chapter playtest:** `scenario-playtest-scenarios:03-combat-deep-dive`
**Relevant skill:** `/inspect`, `/strategy-analysis`, `/balance-sim`

### 🔑 Canonical chain + charge design (user-confirmed 2026-04-08)

This chapter establishes new rules. Implement these precisely; update `.claude/rules/game-conventions.md`, `docs/mechanics/cards.md`, `docs/mechanics/combat.md`, and `docs/GAME_DESIGN.md` in the same commits.

1. **Chain colour locks at the start of each turn** and stays fixed for that turn.
2. **Chain colour is randomized at turn start, weighted by the player's deck composition.**
   - Count each chain type in the player's current deck.
   - Probability of selecting a chain type is proportional to its share of the deck.
   - Example: deck has 13 A / 10 B / 1 C → A = 13/24, B = 10/24, C = 1/24. C is still possible, just rare.
   - Use a seeded RNG so combat replays are deterministic.
3. **Charging a card that matches the current chain colour costs base AP** (no surcharge).
4. **Charging a card that does NOT match the current chain colour costs base AP + surcharge.**
   - Surcharge value remains as configured in game-conventions (currently +1).
   - Off-colour charging is allowed — it's a strategic trade.
5. **Off-colour charging does NOT break the chain multiplier** as long as the answer is correct.
   - If the player answers the quiz correctly, the chain multiplier continues to grow.
   - If the player answers **wrong**, the chain multiplier is **reduced** (not zeroed) — define exact reduction in Pass 2 research. My default suggestion: ×0.5 to current multiplier, floored at ×1.0.
6. **Surge turns waive all charge surcharges** — during a surge turn, both on-colour AND off-colour charges cost base AP only.
