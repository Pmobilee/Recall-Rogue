**NEVER generate distractors programmatically from database pools. EVER.**

This means:
- NEVER query `SELECT correct_answer FROM facts WHERE category = ...` and use results as distractors
- NEVER build "distractor pools" by collecting answers from the same domain/subcategory
- NEVER use hardcoded lists (countries, species, years) as distractor sources
- NEVER write or run scripts like `regen-distractors.mjs`, `mine-distractors.mjs`, or `fix-bad-distractors.mjs`
- NEVER suggest this approach to the user or to sub-agents

**Why:** On March 12, 2026, pool-based distractor generation produced 58,359 garbage distractors that had to be mass-stripped. The approach creates semantically nonsensical wrong answers — bird species names for bird *behavior* questions, random capitals for *flag* questions, "Bengal" for "Where was tea first cultivated?". These are trivially recognizable as wrong and teach nothing. The user considers this a company-threatening issue.

**How to apply:** ALL distractors must be generated by an LLM (GPT-5.2+ sub-worker, or Haiku agent via Claude Code) that:
1. Reads the specific question and correct answer
2. Understands what is being asked (not just the category)
3. Generates plausible wrong answers from world knowledge
4. Matches the format and length of the correct answer
5. Produces answers a real student would actually consider

The ONLY permitted use of DB queries for distractors is POST-GENERATION VALIDATION — checking that a generated distractor doesn't accidentally match another fact's correct answer.

**Banned scripts (delete on sight):**
- `scripts/content-pipeline/regen-distractors.mjs`
- `scripts/content-pipeline/fix-bad-distractors.mjs`
- `scripts/content-pipeline/_DEPRECATED_mine-distractors.mjs`
- Any new script that builds "pools" from DB answers

**When spawning ANY sub-agent that touches distractors**, include this instruction:
> "NEVER generate distractors from database pools or queries. ALL distractors must come from LLM world knowledge. Read the question, understand it, and generate plausible wrong answers. NO pool-based generation."
