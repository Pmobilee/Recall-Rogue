# Playtest Results Viewer

Display the current state of playtest results without running new simulations.

> Note: Data is generated by `/headless-playtest`. Run that first if no data exists.

## Arguments
- `leaderboard` (default): Show the ranked issue leaderboard
- `runs`: List available run directories in `data/playtests/runs/`
- `logs`: List recent playtest logs with summaries
- `report <id>`: Show a specific analysis report
- `log <id>`: Show details of a specific run log
- `engagement`: Show engagement and fun score summary across recent runs

## Steps

### Default: Show Leaderboard
1. Run `node tests/playtest/view-leaderboard.mjs`
2. If no leaderboard exists, tell the user to run `/headless-playtest` first

### `runs`: List Available Run Directories
1. List directories in `data/playtests/runs/` sorted by name (newest timestamp last)
2. For each directory, show:
   ```
   data/playtests/runs/2026-03-21T14-32-00/
   data/playtests/runs/2026-03-21T18-55-12/   ← most recent
   ```
3. If empty, tell the user to run `/headless-playtest` first

### `logs`: List Recent Logs
1. List `combined.json` files under `data/playtests/runs/` directories, sorted newest first (up to 10)
2. For each log, read the JSON and show a summary row per profile present. The SimRunResult schema fields are:
   - `runId`, `profile`, `survived`, `floorsReached`, `accuracy`, `totalCardsPlayed`, `ascensionLevel`
   ```
   Run: data/playtests/runs/2026-03-21T18-55-12/combined.json
   Profile         | Result  | Floors | Accuracy | Cards Played | Ascension
   ----------------|---------|--------|----------|--------------|----------
   first_timer     | defeat  |      3 |   61.2%  |           42 |         0
   casual_learner  | defeat  |      5 |   72.4%  |           67 |         0
   regular         | victory |      8 |   80.1%  |           98 |         0
   gamer           | victory |     12 |   85.3%  |          134 |         0
   dedicated       | victory |     16 |   88.7%  |          158 |         0
   scholar         | victory |     24 |   93.1%  |          201 |         0
   ```

### `report <id>`: Show Report
1. Find the matching file in `data/playtests/reports/`
2. Read and display the issues found:
   ```
   Report: report-2026-03-21T18-55-12.json
   Issues found: 3

   1. [HIGH] balance_damage_spike — Floor 5 boss deals 45 damage in one turn
      Evidence: Turn 89, Floor 5, Encounter 2
      Metric: Enemy dealt 45 damage (56% of 80 HP)

   2. [MEDIUM] balance_combo_unreachable — Max combo never reaches 4 despite 87.8% accuracy
      Evidence: Across all encounters
   ```

### `log <id>`: Show Log Details
1. Find the matching `combined.json` under `data/playtests/runs/<id>/`
2. The SimRunResult schema top-level fields:
   - `runId`, `totalTurns`, `totalDamageDealt`, `totalDamageTaken`, `totalCardsPlayed`
   - `totalCorrect`, `totalWrong`, `accuracy`, `floorsReached`, `finalHP`, `survived`, `ascensionLevel`
3. Each entry in `encounters` has:
   - `encounterIndex`, `floor`, `enemyId`, `enemyName`, `result`, `turnsUsed`
   - `damageDealtTotal`, `damageTakenTotal`, `cardsPlayed`, `correctAnswers`, `wrongAnswers`
   - `maxCombo`, `playerHpStart`, `playerHpEnd`
4. Show:
   ```
   Run: 2026-03-21T18-55-12 | Profile: gamer | Ascension: 0
   Result: victory | Floors: 12 | Final HP: 34/80
   Turns: 156 | Cards: 134 | Accuracy: 85.3% (102C / 18W)
   Damage Dealt: 1240 | Damage Taken: 460

   Encounter Breakdown:
   Floor | Enc | Enemy           | Result  | Turns | HP End | Accuracy | MaxCombo
   ------|-----|-----------------|---------|-------|--------|----------|--------
     1   |  0  | cave_bat        | victory |     3 |     75 |  100.0%  |       3
     1   |  1  | crystal_golem   | victory |     4 |     62 |   83.3%  |       2
     2   |  0  | shadow_sentinel | victory |     5 |     48 |   80.0%  |       3
   ```

### `engagement`: Show Engagement Summary
1. List the 10 most recent `combined.json` files under `data/playtests/runs/`
2. For each, read the JSON and extract `summary.deepStats.engagement` (if present)
3. Show:
   ```
   Engagement & Fun Scores — Recent Runs

   Profile          | Seed  | Floor | Engagement | Fun  | Dead% | Slogs | Streaks (C/W) | Facts | Trend
   -----------------|-------|-------|------------|------|-------|-------|---------------|-------|------
   scholar          | 42    |    24 |         71 |   74 |  2.1% |     0 |        8 / 2  |   142 |  +0.05
   dedicated        | 42    |    16 |         63 |   61 |  3.4% |     0 |        7 / 3  |   118 |  +0.03
   gamer            | 42    |    12 |         55 |   52 |  5.3% |     1 |        5 / 4  |    87 |  +0.02
   regular          | 42    |     8 |         47 |   45 |  7.1% |     1 |        4 / 5  |    66 |  -0.01
   casual_learner   | 42    |     5 |         38 |   35 |  9.8% |     2 |        3 / 6  |    48 |  -0.05
   first_timer      | 42    |     3 |         28 |   24 | 14.2% |     3 |        2 / 7  |    31 |  -0.10

   Averages:        Engagement: 50/100 | Fun: 49/100

   Runs with engagement < 30: 1 (first_timer)
   Runs with fun < 30: 1 (first_timer)
   ```
4. If no runs have deepStats.engagement, tell the user to re-run playtests with deep mode enabled
