# Lobby Test Action Sequences

Pre-built action sequences for the 10 lobby test scenarios. Each block shows the host actions, guest actions, state extraction, and pass criteria.

All selectors use `data-testid` attributes added to the MP components. Fallback selectors are provided where fragility is a concern.

---

## Common Setup

Before each scenario that needs a fresh lobby, navigate both containers to hub:

```json
[
  {"type": "rrPlay", "method": "navigate", "args": ["hub"]},
  {"type": "wait", "ms": 1000}
]
```

Player IDs for the session:
- Host: `mp-test-host-001`
- Guest: `mp-test-guest-001`

These IDs are generated by `generatePlayerId()` in the lobby service when the game boots. The eval actions below read the actual IDs from the running game.

## S1: Create Lobby + Code Generation

### Host Actions

```json
[
  {"type": "rrPlay", "method": "navigate", "args": ["multiplayerMenu"]},
  {"type": "wait", "ms": 1500},
  {"type": "screenshot", "name": "S1-menu"},
  {"type": "click", "selector": "[data-testid='mode-race']", "waitAfter": 500},
  {"type": "click", "selector": "[data-testid='btn-create-lobby']", "waitAfter": 2000},
  {"type": "screenshot", "name": "S1-lobby-created"},
  {"type": "eval", "js": "JSON.stringify({code: document.querySelector('[data-testid=\"lobby-code\"]')?.textContent?.trim(), screen: window.__rrPlay?.getScreen()})"}
]
```

### State Extraction

From host `result.json` eval output, extract `code`. Also verify via REST:
```bash
curl -s http://localhost:3000/mp/lobbies | python3 -c "import sys,json; d=json.load(sys.stdin); print(json.dumps(d.get('lobbies',[])))"
```

### Pass Criteria
- `code` is a 6-character string matching `[A-Z2-9]{6}`
- `screen` is `multiplayerLobby`
- Fastify `/mp/lobbies` returns 1 lobby with matching code

## S3: Password Protection

### Host Actions (create password lobby)

First leave any existing lobby by navigating to hub, then:

```json
[
  {"type": "rrPlay", "method": "navigate", "args": ["multiplayerMenu"]},
  {"type": "wait", "ms": 1000},
  {"type": "click", "selector": "[data-testid='mode-race']", "waitAfter": 500},
  {"type": "click", "selector": "[data-testid='btn-create-lobby']", "waitAfter": 2000},
  {"type": "click", "selector": "[data-testid='visibility-toggle']", "waitAfter": 500},
  {"type": "screenshot", "name": "S3-visibility-options"}
]
```

After clicking the visibility toggle, select "Password" option (the exact selector depends on the dropdown/radio implementation -- use eval to inspect):

```json
[
  {"type": "eval", "js": "JSON.stringify({visOptions: [...document.querySelectorAll('[class*=visibility], [class*=vis-option], select option')].map(e => e.textContent?.trim())})"}
]
```

Then set password:
```json
[
  {"type": "fill", "selector": "[data-testid='password-input']", "text": "test123"},
  {"type": "wait", "ms": 500},
  {"type": "screenshot", "name": "S3-password-set"},
  {"type": "eval", "js": "JSON.stringify({code: document.querySelector('[data-testid=\"lobby-code\"]')?.textContent?.trim()})"}
]
```

### Guest Actions (join without password -- expect failure)

```json
[
  {"type": "rrPlay", "method": "navigate", "args": ["multiplayerMenu"]},
  {"type": "wait", "ms": 1000},
  {"type": "click", "selector": "[data-testid='tab-join']", "waitAfter": 500},
  {"type": "fill", "selector": "[data-testid='join-code-input']", "text": "{LOBBY_CODE}"},
  {"type": "click", "selector": "[data-testid='btn-join-lobby']", "waitAfter": 2000},
  {"type": "screenshot", "name": "S3-guest-no-password"},
  {"type": "eval", "js": "JSON.stringify({screen: window.__rrPlay?.getScreen(), errorVisible: document.querySelector('[class*=error], [role=alert]')?.textContent?.trim() || null})"}
]
```

**Note**: The join-by-code path may prompt for a password if the lobby is password-protected. Check whether the UI shows an error or a password prompt. Either way, verify the guest is NOT in the lobby.

### Guest Actions (join with password -- expect success)

If the UI shows a password modal:
```json
[
  {"type": "fill", "selector": "[data-testid='password-modal-input'], input[type='password']", "text": "test123"},
  {"type": "click", "selector": "[data-testid='btn-password-submit'], button:has-text('Join')", "waitAfter": 2000},
  {"type": "screenshot", "name": "S3-guest-with-password"},
  {"type": "eval", "js": "JSON.stringify({screen: window.__rrPlay?.getScreen(), players: document.querySelectorAll('[data-testid^=\"player-slot-\"]').length})"}
]
```

### Pass Criteria
- Join without password: guest is NOT on `multiplayerLobby` (or an error/password prompt is shown)
- Join with password: guest is on `multiplayerLobby`, 2 players visible

## S5: Deck Selection

### Prerequisites
Both players in lobby (S1+S2 or fresh).

### Host Actions

```json
[
  {"type": "click", "selector": "[data-testid='btn-deck-picker']", "waitAfter": 1500},
  {"type": "screenshot", "name": "S5-deck-picker-open"},
  {"type": "eval", "js": "JSON.stringify({hasPicker: document.querySelector('[class*=deck-picker], [class*=DeckPicker]') !== null, deckOptions: document.querySelectorAll('[class*=deck-item], [class*=deck-card], [class*=deck-row]').length})"}
]
```

Select the first available deck (click first deck item):
```json
[
  {"type": "click", "selector": "[class*=deck-item]:first-child, [class*=deck-card]:first-child, [class*=deck-row]:first-child", "waitAfter": 500},
  {"type": "screenshot", "name": "S5-deck-selected"},
  {"type": "eval", "js": "JSON.stringify({selectedDeck: document.querySelector('[class*=selected][class*=deck]')?.textContent?.trim()?.slice(0, 50)})"}
]
```

Close picker and verify on guest:
```json
[
  {"type": "key", "key": "Escape", "waitAfter": 500}
]
```

### Guest Verification (after deck selection)

```json
[
  {"type": "wait", "ms": 2000},
  {"type": "eval", "js": "JSON.stringify({lobbyContent: document.querySelector('[class*=content-selection], [class*=deck-display]')?.textContent?.trim()?.slice(0, 100)})"},
  {"type": "screenshot", "name": "S5-guest-sees-deck"}
]
```

### Pass Criteria
- Deck picker opens on host
- A deck can be selected
- Guest's lobby state shows the same content selection (check via REST: `GET /mp/lobbies` and inspect `contentSelection`)

## S7: Max Players (Server-Side Enforcement)

### Prerequisites
Lobby at capacity (2 players for race mode).

### REST Verification

Extract lobbyId from Fastify:
```bash
LOBBY_ID=$(curl -s http://localhost:3000/mp/lobbies | python3 -c "import sys,json; print(json.load(sys.stdin)['lobbies'][0]['lobbyId'])")
```

Attempt 3rd join via REST:
```bash
HTTP_CODE=$(curl -s -o /tmp/mp-join-response.json -w "%{http_code}" \
  -X POST "http://localhost:3000/mp/lobbies/${LOBBY_ID}/join" \
  -H "Content-Type: application/json" \
  -d '{"playerId":"player-3-overflow","displayName":"Player 3"}')
echo "HTTP_CODE=$HTTP_CODE"
cat /tmp/mp-join-response.json
```

### Pass Criteria
- HTTP response is 409 (Conflict) or 403 (Forbidden)
- Error message indicates lobby is full

## S8: Visibility States

### REST Verification

For each visibility state, verify listing behavior:

**Public** (default): `GET /mp/lobbies` returns the lobby.

**Password**: After host sets visibility to `password`:
- `GET /mp/lobbies` returns the lobby with `hasPassword: true`

**Friends-only**: After host sets visibility to `friends_only`:
- `GET /mp/lobbies` returns empty (friends_only lobbies are filtered out on web/broadcast)

### Pass Criteria
- Public: listed, `hasPassword: false`
- Password: listed, `hasPassword: true`
- Friends-only: NOT listed
