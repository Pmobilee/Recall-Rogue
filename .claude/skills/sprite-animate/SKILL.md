# Sprite Animate — Idle Animation Generator

## Animation Spec Reference

**`docs/ENEMY-ANIMATIONS.md`** — Complete animation spec for all 87 active enemies. Contains:
- Per-enemy animation lists (up to 8 + idle) based on actual intent pools and special mechanics
- Full LTX-2.3-ready prompts with lore-appropriate motion descriptions
- Animation categories: idle, attack, multi_attack, defend, buff, debuff, heal, charge, hit, death, phase2, special
- Organized by Act (1-3) and enemy category (common/mini-boss/elite/boss)

**Always consult this doc** when generating enemy animations to use the correct prompt for the specific enemy and animation type.

## HARD RULES — READ BEFORE ANYTHING ELSE

### Sprite Sources
- **Enemy sprites for LTX animation** MUST come from `sprite-gen/cardback-tool/artstudio-output/enemies-accepted/` — 87 files, 1024x1024 RGBA. This is the ONLY permitted source. NEVER use sprites from `public/assets/`, `src/assets/`, or any other location.
- **Icons, relics, and other non-enemy sprites** MAY be generated via ComfyUI (SDXL + pixel-art-xl LoRA) as needed.
- The old `data/generated/enemies/` folder is ARCHIVED and must NEVER be used.

### Prompt Rules — MANDATORY
LTX generates video from your prompt + the input image. Since we cut out the enemy frame-by-frame via background removal, the prompt MUST follow these rules:

- **Keep prompts extremely simple** — describe ONLY the motion/morphing of existing pixels
- **NO color flashing, glowing, pulsing, fading, or light effects** — color changes get cut out during background removal, leaving holes
- **NO adding elements** — no sparks, particles, debris, dust, extra limbs, auras, clouds, shards, or anything outside the original sprite's pixels. These cause awful artifacts when cut out
- **NO dissolving, evaporating, or transparency changes** — the sprite must remain solid throughout
- **Just morph the existing shape** — tilt, slump, collapse, squash, stretch, rotate, shrink. The original pixels move, they don't change color or spawn new things
- Death animations: the creature slumps, collapses, or crumples. It does NOT explode, shatter, dissolve, or fade away
- Keep the style line but remove any mention of glowing, energy, or effects from it

## Pipeline Overview

1. Upload sprite to ComfyUI input
2. Read original sprite dimensions to determine aspect-ratio-aware generation size (default target: 768px longer edge)
3. Submit LTX-2.3 I2V workflow at computed resolution, 3s @ 24fps = 73 frames
4. Poll until generation complete (~3-5 min on M4 Max MPS)
5. Download individual PNG frames
6. Remove background per-frame with rembg using `birefnet-general` model (higher quality than u2net)
7. Resize each frame back to original sprite dimensions using LANCZOS resampling (no padding crop needed)
8. Encode transparent APNG (primary format) + WebM preview + MP4 preview
9. Generate horizontal spritesheet PNG (all frames, with alpha)

## Prompt Engineering Guide

**Template:**
```
A [detailed creature description matching the sprite] facing forward on a solid black background. Idle breathing animation loop.

0-1 seconds: [Primary idle motion — breathing, body rise/fall, energy pulse]. [Creature-specific detail].

1-2 seconds: [Secondary motion — appendage twitch, eye movement, weight shift]. [Brief heightened activity].

2-3 seconds: Returns smoothly to starting pose for seamless loop. Breathing rhythm matches frame 0.

Style: Pixel art sprite animation, limited frame-rate 8-12 fps choppy movement, no environment, no ground texture, sprite centered and stationary, no camera movement, locked tripod, no zoom, no pan, solid black background.
```

**Tips:**
- Describe the sprite visually first — LTX-2.3 uses both the image AND text to generate
- Keep motions subtle — large movements will distort the sprite
- Always end with "returns to starting pose for seamless loop"
- The "solid black background" is critical for clean background removal
- Mention specific visual features (crystals glowing, flames flickering, eyes scanning)

**CRITICAL — What NOT to include in prompts:**
- No glowing, pulsing, flashing, or color changes
- No particles, sparks, debris, dust, smoke, or clouds
- No extra limbs, appendages, or elements not in the original sprite
- No dissolving, fading, evaporating, or shattering into pieces
- These all cause artifacts during frame-by-frame background removal

**Negative prompt (hardcoded):**
Camera movement, zoom, pan, tilt, background scenery, realistic, 3D render, smooth animation, environment, floor texture, shadows on background, morphing shape, extra limbs, deformation, melting

## Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| --sprite | required | Path to input sprite PNG (MUST be from `sprite-gen/cardback-tool/artstudio-output/enemies-accepted/`) |
| --prompt | auto-gen | Animation description |
| --duration | 3 | Seconds (2-5 recommended) |
| --target-size | 768 | Max dimension (longer edge) for generation; aspect ratio auto-calculated from original sprite |
| --width | auto | Override generation width explicitly (bypasses --target-size) |
| --height | auto | Override generation height explicitly (bypasses --target-size) |
| --rembg-model | birefnet-general | Background removal model; alternatives: `u2net`, `isnet-general-use` |
| --seed | random | Reproducibility seed |
| --name | from filename | Output naming |
| --output-dir | sprite-gen/output/animations | Output location |
| --comfyui-url | http://localhost:8188 | ComfyUI server URL |
| --skip-generate | false | Skip ComfyUI, just post-process existing raw-frames |
| --skip-postprocess | false | Skip bg removal + encoding, only generate frames |

## Output Files


## Troubleshooting

| Problem | Solution |
|---------|----------|
| OOM during generation | Lower target size: `--target-size 512`, or reduce duration to 2s |
| Models fail to load | Ensure ComfyUI loaded without errors. Check `/tmp/comfyui-startup.log` |
| Black output frames | Sprite wasn't uploaded — check ComfyUI input/ directory |
| Background removal artifacts | Try `--rembg-model isnet-general-use` or `--rembg-model u2net` |
| Edge halos after removal | birefnet-general is generally cleanest; try `u2net_human_seg` for humanoid shapes |
| "Address already in use" | Kill existing: `lsof -ti:8188 | xargs kill -9` then restart |
| NAG node error | NAG is bypassed in our workflow (GGUF incompatibility). This is expected. |

## Technical Details

- **Models**: LTX-2.3 22B distilled Q4_K_S GGUF (~16GB) + Gemma 3 12B fpmixed (~13.7GB)
- **ComfyUI path**: /Users/damion/CODE/ComfyUI/
- **Workflow**: sprite-gen/workflows/ltx-sprite-animate-api.json
- **Script**: sprite-gen/scripts/animate-sprite.mjs
- **VRAM usage**: ~30GB peak (tight on 36GB unified memory)
- **Generation time**: ~3-5 min per sprite on M4 Max MPS
- **Frame count formula**: `1 + 8 * round(duration * fps / 8)` → 73 frames for 3s@24fps
- **Default bg removal model**: birefnet-general (weights at `/Users/damion/.u2net/birefnet-general.onnx`)
- **Aspect ratio handling**: Script reads original sprite dimensions, calculates generation size to fit within `--target-size` while preserving ratio (rounded to nearest 32px per LTX requirement), then resizes output frames back to original dimensions using LANCZOS

## Integration with Phaser

The spritesheet PNG can be loaded directly (frame dimensions match original sprite size):
```typescript
this.load.spritesheet('enemy_idle', 'assets/sprites/enemies/rock_hermit_idle_sheet.png', {
  frameWidth: origWidth,   // e.g. 922 for rock_hermit
  frameHeight: origHeight  // e.g. 573 for rock_hermit
});
```
WebM video textures are also possible via `this.load.video()` but spritesheet is more reliable cross-platform.
